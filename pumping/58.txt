<h1 class="heading__title">Why Page Speed Matters: 10 Case Studies Show How</h1>
<div class="heading__params">
<span aria-hidden="true" class="heading__label">
										Last updated:
									</span>
<span aria-label="The date of last update" class="heading__caption">

<div class="post-detail__text typo">
<section aria-label="Why Page Speed Matters: 10 Case Studies" class="highlighted-box highlighted-box--with-background typo-block"><div aria-hidden="true" class="highlighted-box__title">
<span class="highlighted-box__title-icon"><i class="post-type-icon post-type-icon--reversed-style" style="background-image: url(/assets/images/icons/post-types/page-speed-resources--reversed.svg)"></i></span>Why Page Speed Matters: 10 Case Studies
	</div>
<div class="highlighted-box__content">
<p>Page speed matters, especially with Core Web Vitals now being a ranking factor for Google. Case studies from sites like Vodafone, Yelp and Ebay show how even milliseconds can impact bounce rates, conversion rates, and ultimately your bottom line.</p>
</div>
</section>

<div aria-hidden="true" class="highlighted-box__title">Table of contents</div>

<p>Having a fast-loading site is important for the customer experience, and as such, important for making money. Think about your own experience as a user — are you going to still stay on a site if a spinning wheel appears, or will you bounce back and find another that gets you to what you want more quickly? <a href="https://www.thinkwithgoogle.com/marketing-strategies/app-and-mobile/mobile-page-speed-new-industry-benchmarks/" rel="noopener" target="_blank"><span class="extlink">For most people</span><span class="show-for-sr"> (opens in a new tab)</span></a>, the faster site will be the winner.</p>
<p>And with Core Web Vitals <a href="https://developers.google.com/search/blog/2020/11/timing-for-page-experience" rel="noopener" target="_blank"><span class="extlink">now becoming</span><span class="show-for-sr"> (opens in a new tab)</span></a> a ranking factor for Google Search, speed matters more than ever.</p>
<p>But just <em>how important</em> is page speed? And how much can it impact your bottom line?</p>
<p>To help answer this, we’ve compiled the latest page speed case studies that highlight the importance and benefits of having a fast site. The case studies span across several industries and business sizes, as well as feature a variety of different performance related metrics, including Core Web Vitals.</p>
<h2 id="a-primer-on-core-web-vitals">
<a aria-hidden="true" class="anchor-link" href="#a-primer-on-core-web-vitals" tabindex="-1"></a>A primer on Core Web Vitals</h2>
<p>But before we dive in, it’s important that we cover <a href="/academy/core-web-vitals/">Core Web Vitals’</a> because of their roles in the case studies we cover below.</p>
<p>Core Web Vitals are a set of user-facing metrics related to speed, responsiveness and visual stability, to help site owners measure user experience on the web.</p>
<p><a href="/blog/page-experience/">Google’s Page Experience Update</a> merges Core Web Vitals with existing User Experience ranking factors to form a new ranking factor category called "Search signals for page experience". This means Core Web Vitals will now be a ranking factor for Google.</p>
<p>The Web Vitals metrics are split into Core Web Vitals and non-Core Web Vitals.</p>
<p>The Core Web Vitals are:</p>
<ul>
<li>
<a href="/academy/core-web-vitals/#lcp">Largest Contentful Paint (LCP)</a>: measures the time in seconds from when the page starts loading to when the largest text block or image element is rendered on the screen.</li>
<li>
<a href="/academy/core-web-vitals/#fid">First Input Delay (FID)</a>: measures the time in milliseconds from when a user first interacts with your site to when the browser is able to respond to that interaction.</li>
<li>
<a href="/academy/core-web-vitals/#cls">Cumulative Layout Shift (CLS)</a>: measures the cumulative score of all unexpected layout shifts within the viewport that occur during a page’s entire lifecycle.</li>
</ul>
<p>And the non-Core Web Vitals are:</p>
<ul>
<li>
<a href="/academy/core-web-vitals/#tbt">Total Blocking Time (TBT)</a>: measures the total time in milliseconds between First Contentful Paint (FCP) and Time To Interactive (TTI) where the main thread is blocked long enough to make it unresponsive to user input.</li>
<li>
<a href="/academy/core-web-vitals/#fcp">First Contentful Paint (FCP)</a>: measures the time from when a page starts loading to when any part of that page’s content is rendered on the screen.</li>
<li>
<a href="/academy/core-web-vitals/#si">Speed Index (SI)</a>: measures how quickly the contents of a page are visibly populated during page load.</li>
<li>
<a href="/academy/core-web-vitals/#tti">Time to Interactive (TTI)</a>: measures the time from when the page starts loading to when it's fully interactive.</li>
</ul>
<p>As we mentioned before, Core Web Vitals are a ranking factor for Google Search as part of the new page experience signals. And next to that, it’s always important to provide users with a good experience on your site.</p>
<p>So now that we’ve covered Core Web Vitals, let’s check out the case studies!</p>
<h2 id="google">
<a aria-hidden="true" class="anchor-link" href="#google" tabindex="-1"></a>1. Google: Page load speed increases bounce rate by 123%</h2>
<ul>
<li>
<strong>Year published:</strong> 2017 (updated in 2018)</li>
<li>
<strong>Metrics covered:</strong> Load speed, bounce rate, conversion rate</li>
<li>
<strong>Read here:</strong> <a href="https://www.thinkwithgoogle.com/marketing-strategies/app-and-mobile/mobile-page-speed-new-industry-benchmarks/" rel="noopener" target="_blank"><span class="extlink">Think with Google</span><span class="show-for-sr"> (opens in a new tab)</span></a>
</li>
</ul>
<p>Google did an analysis of 11 million mobile ads’ landing pages spanning across 213 countries, and found that a majority of mobile sites were too slow for users' expectations. For each second it took for a page to load, the probability of a mobile site user bouncing increased — and as a result the probability of conversion decreases, even dropping as dramatically as 95%.</p>

<h2 id="vodafone">
<a aria-hidden="true" class="anchor-link" href="#vodafone" tabindex="-1"></a>2. Vodafone: 8% more sales by optimizing Core Web Vitals</h2>
<ul>
<li>
<strong>Year published:</strong> 2021</li>
<li>
<strong>Metrics covered:</strong> Visitor rate, visitor to cart rate, conversion rate</li>
<li>
<strong>Read here:</strong> <a href="https://web.dev/vodafone/" rel="noopener" target="_blank"><span class="extlink">Web.dev</span><span class="show-for-sr"> (opens in a new tab)</span></a>
</li>
</ul>
<p>Running an evenly weighed A/B test on a landing page, Vodafone found that optimizing Core Web Vitals had a positive impact on the customer journey. The test measured two visually and functionally identical pages, the only difference was the optimized page had a 31% better LCP score than the baseline page. The results showed that optimized landing page had an 8% increase in sales.</p>
<section aria-label="Why this is important" class="remark-box typo-block"><div aria-hidden="true" class="remark-box__title">Why this is important</div>
<div class="remark-box__text">
<p>Having a fast LCP tells users that a page is useful, and as a result, keeps them on your site. There’s several factors that potentially could be hurting your LCP. <a href="/academy/core-web-vitals/#what-could-be-causing-a-poor-lcp-score">Learn what causes a poor LCP score.</a></p>
</div></section>
<h2 id="yelp">
<a aria-hidden="true" class="anchor-link" href="#yelp" tabindex="-1"></a>3. Yelp: Optimizing FCP boosted conversions by 15%</h2>
<ul>
<li>
<strong>Year published:</strong> 2021</li>
<li>
<strong>Metrics covered:</strong> Conversion rate</li>
<li>
<strong>Read here:</strong> <a href="https://engineeringblog.yelp.com/2021/01/boosting-user-conversion-with-ux-performance-wins.html" rel="noopener" target="_blank"><span class="extlink">Yelp</span><span class="show-for-sr"> (opens in a new tab)</span></a>
</li>
</ul>
<p>Yelp recently added new features for advertisers to have more control over their ad campaigns. But when they added these new features, they compromised speed with page load times increasing from 3 seconds to 6 seconds. The development team combatted this slowdown by optimizing FCP and TTI metrics, resulting in a site that provides both a great UX experience while also boosting conversions by 15%.</p>
<section aria-label="Why this is important" class="remark-box typo-block"><div aria-hidden="true" class="remark-box__title">Why this is important</div>
<div class="remark-box__text">
<p>If your site's FCP score is &gt; 3s, it's considered slow. High server response times and render-blocking resources are common causes for a poor FCP score. <a href="/academy/core-web-vitals/#improving-your-fcp-score">Read about what you can do to improve FCP.</a></p>
</div></section>
<h2 id="akamai">
<a aria-hidden="true" class="anchor-link" href="#akamai" tabindex="-1"></a>4. Akamai: 100-millisecond delay hurt conversion rates by 7%</h2>
<ul>
<li>
<strong>Year published:</strong> 2019</li>
<li>
<strong>Metrics covered:</strong> Load speed, bounce rate, conversion rate</li>
<li>
<strong>Read here:</strong> <a href="https://www.akamai.com/uk/en/about/news/press/2017-press/akamai-releases-spring-2017-state-of-online-retail-performance-report.jsp" rel="noopener" target="_blank"><span class="extlink">Akamai</span><span class="show-for-sr"> (opens in a new tab)</span></a>
</li>
</ul>
<p>Akamai analyzed data from over 10 billion visits to top retail sites and found that even milliseconds matter. The case study shows that even the smallest amount of time can lead to customers going elsewhere, with over half of mobile shoppers leaving a page if it takes longer than just three seconds. Even delays down to the millisecond can lower conversion rates.</p>

One of the challenges of today's web where JavaScript increases.</p>

<h2 id="ebay">
<a aria-hidden="true" class="anchor-link" href="#ebay" tabindex="-1"></a>5. Ebay: Better ROI with 100ms improvement in load time</h2>
<ul>
<li>
<strong>Year published:</strong> 2020</li>
<li>
<strong>Metrics covered:</strong> Conversion rate</li>
<li>
<strong>Read here:</strong> <a href="https://tech.ebayinc.com/engineering/speed-by-a-thousand-cuts/" rel="noopener" target="_blank"><span class="extlink">Ebay Engineering Blog</span><span class="show-for-sr"> (opens in a new tab)</span></a> and <a href="https://web.dev/shopping-for-speed-on-ebay/" rel="noopener" target="_blank"><span class="extlink">Web.dev</span><span class="show-for-sr"> (opens in a new tab)</span></a>
</li>
</ul>
<p>Ebay set forth a company-wide initiative focused primarily on above-the-fold content. They used metrics like Time To First Byte (TTFB) and Time to Above The Fold (TATF) to guide their efforts — however optimizations were only possible if certain parts of the customer journey were cut. By making certain cuts and improving above-the-fold content, pages loaded 12% faster and resulted in a 0.5% “Add to Cart” increase.</p>
<section aria-label="Why this is important" class="remark-box typo-block"><div aria-hidden="true" class="remark-box__title">Why this is important</div>
<div class="remark-box__text">
<p>Satisfying a searcher's intent quickly is important for customer experience and SERP rankings. In recent years, this has become a major ranking factor for search engines. <a href="https://web.dev/tti/#how-to-improve-tti" rel="noopener" target="_blank"><span class="extlink">Here’s best practices to help improve Time to Interactive (TTI)</span><span class="show-for-sr"> (opens in a new tab)</span></a>.</p>

<h2 id="propellernet">
<a aria-hidden="true" class="anchor-link" href="#propellernet" tabindex="-1"></a>6. Propellernet: Faster visits 34% more likely to convert</h2>
<ul>
<li>
<strong>Year published:</strong> 2020</li>
<li>
<strong>Metrics covered:</strong> Conversion rate</li>
<li>
<strong>Read here:</strong> <a href="https://www.propellernet.co.uk/the-impact-of-speed-on-ecommerce/" rel="noopener" target="_blank"><span class="extlink">Propellernet</span><span class="show-for-sr"> (opens in a new tab)</span></a>
</li>
</ul>
<p>Propellernet wanted to know if it was really true that a faster site meant higher conversions, and if it was, was it a noticeable difference? This led them to do an experiment using real time monitoring tools to track landing page speed metrics down to the millisecond, on both desktop vs. mobile and iOS vs. Android. And the theory was correct, faster-than-average visits were 34% more likely to convert versus slower-than-average visits.</p>

<p>Mobile visits where FCP was faster-than-average were 32% more likely to convert, while mobile visits where LCP was faster-than-average were 27% more likely to go on to convert.</p>

<h2 id="portent">
<a aria-hidden="true" class="anchor-link" href="#portent" tabindex="-1"></a>8. Portent: 0-4 Seconds is the ideal page load time</h2>
<ul>
<li>
<strong>Year published:</strong> 2019</li>
<li>
<strong>Metrics covered:</strong> Load speed, conversion rate</li>
<li>
<strong>Read here:</strong> <a href="https://www.portent.com/blog/analytics/research-site-speed-hurting-everyones-revenue.htm" rel="noopener" target="_blank"><span class="extlink">Portent</span><span class="show-for-sr"> (opens in a new tab)</span></a>
</li>
</ul>
<p>Portent looked at data from 94 million page views across 10 ecommerce sites, covering  nation-wide retailers to small niche brands in both B2B and B2C. The aim of the study was to find how much speed impacts conversion rates. The results showed that the dropoff for conversions is much steeper after 5 seconds of loading time— making the ideal page load time between 0-4 seconds.</p>
<section aria-label="Why this is important" class="remark-box typo-block"><div aria-hidden="true" class="remark-box__title">Why this is important</div>
<div class="remark-box__text">
<p>Site speed matters for not only users, but also crawl budget. Faster page load time means Google can crawl more pages on your site, and new content can be discovered and indexed quicker. <a href="/academy/crawl-budget/#how-optimize-crawl-budget">Read about how to maximize your crawl budget.</a></p>
