<div class="heading">
<h1 class="heading__title">Setting up the Akamai integration</h1>
</div>
<div class="post-detail__text typo">
<div aria-label="Table of contents" class="highlighted-box" role="navigation">
<div aria-hidden="true" class="highlighted-box__title">Table of contents</div>
<div class="highlighted-box__content">
<ul class="navigation-list">
<li class="navigation-list__item navigation-list__item--primary"><a href="#setting-up-the-log-file-analysis">Setting up the Log File Analysis</a></li>
<li class="navigation-list__item navigation-list__item--secondary"><a href="#1-enabling-the-log-file-analysis-feature-and-creating-an-aws-s3-bucket-in-contentking">1. Enabling the Log File Analysis feature and creating an AWS S3 bucket in ContentKing</a></li>
<li class="navigation-list__item navigation-list__item--secondary"><a href="#2-configuring-the-akamai-datastream">2. Configuring the Akamai DataStream</a></li>
<li class="navigation-list__item navigation-list__item--primary"><a href="#disabling-log-file-analysis">Disabling Log File Analysis</a></li>
</ul>
</div>
</div>
<p>Setting up Log File Analysis with the <a href="https://techdocs.akamai.com/datastream2/docs" rel="noopener" target="_blank"><span class="extlink">Akamai DataStream</span><span class="show-for-sr"> (opens in a new tab)</span></a> in ContentKing is very straightforward! Follow the steps outlined below to enable Log File Analysis in ContentKing, and set up the Akamai integration.</p>
<h2 id="setting-up-the-log-file-analysis">
<a aria-hidden="true" class="anchor-link" href="#setting-up-the-log-file-analysis" tabindex="-1"></a>Setting up the Log File Analysis</h2>
<p>The process of configuring the Log File Analysis consists of two phases:</p>
<ol>
<li>
<p>Enabling the Log File Analysis feature and creating an <a href="https://aws.amazon.com/s3/" rel="noopener" target="_blank"><span class="extlink">AWS S3 bucket</span><span class="show-for-sr"> (opens in a new tab)</span></a> in ContentKing</p>
</li>
<li>
<p>Configuring the Akamai DataStream in the Akamai UI and associating the DataStream to the corresponding property</p>
</li>
</ol>
<h3 class="toc" id="1-enabling-the-log-file-analysis-feature-and-creating-an-aws-s3-bucket-in-contentking">
<a aria-hidden="true" class="anchor-link" href="#1-enabling-the-log-file-analysis-feature-and-creating-an-aws-s3-bucket-in-contentking" tabindex="-1"></a>1. Enabling the Log File Analysis feature and creating an AWS S3 bucket in ContentKing</h3>
<p>If you navigate to <code class="language-plaintext highlighter-rouge">Account</code> and then <code class="language-plaintext highlighter-rouge">Websites</code> section in ContentKing, you can easily filter and see which of the websites you are monitoring in ContentKing are on Akamai:</p>
<div class="media-container" style="max-width: 1200px;"><picture><source media="(max-width: 600px)" srcset="/media/support/contentking-lfa-websites@1x.avif" type="image/avif"/><source srcset="/media/support/contentking-lfa-websites@1x.avif, /media/support/contentking-lfa-websites@2x.avif 2x" type="image/avif"/><source media="(max-width: 600px)" srcset="/media/support/contentking-lfa-websites@1x.webp" type="image/webp"/><source srcset="/media/support/contentking-lfa-websites@1x.webp, /media/support/contentking-lfa-websites@2x.webp 2x" type="image/webp"/><source media="(max-width: 600px)" srcset="/media/support/contentking-lfa-websites@1x.png"/><img alt="Screenshot of the website section in ContentKing" class="image" height="750" src="/media/support/contentking-lfa-websites@1x.png" srcset="/media/support/contentking-lfa-websites@1x.png, /media/support/contentking-lfa-websites@2x.png 2x" title="Checking eligible Akamai websites in ContentKing" width="1200"/></picture></div>
<p>From there, just follow the steps outlined below to enable Log File Analysis on the wanted website:</p>
<section class="highlighted-box typo-block highlighted-box--with-icon highlighted-box--steps-style"><div class="highlighted-box__content">
<ol>
<li>Click any website that is running on Akamai in the <a href="https://app.contentkingapp.com/account/websites?view=list" rel="noopener" target="_blank"><span class="extlink">Websites section</span><span class="show-for-sr"> (opens in a new tab)</span></a> of ContentKing</li>
<li>Click on <code class="language-plaintext highlighter-rouge">Log File Analysis</code> tab in settings</li>
<li>Move the Log File Analysis toggle to <code class="language-plaintext highlighter-rouge">Enabled</code> state</li>
</ol>
</div></section>
<p>After this is done, follow the next steps to create an <em>AWS S3</em> bucket in ContentKing:</p>
<ul>
<li>In the same Log File Analysis section, click on <code class="language-plaintext highlighter-rouge">How to install</code> link under the Log Sources</li>
</ul>
<div class="media-container" style="max-width: 1200px;"><picture><source media="(max-width: 600px)" srcset="/media/support/contentking-lfa-akamai-install-button@1x.avif" type="image/avif"/><source srcset="/media/support/contentking-lfa-akamai-install-button@1x.avif, /media/support/contentking-lfa-akamai-install-button@2x.avif 2x" type="image/avif"/><source media="(max-width: 600px)" srcset="/media/support/contentking-lfa-akamai-install-button@1x.webp" type="image/webp"/><source srcset="/media/support/contentking-lfa-akamai-install-button@1x.webp, /media/support/contentking-lfa-akamai-install-button@2x.webp 2x" type="image/webp"/><source media="(max-width: 600px)" srcset="/media/support/contentking-lfa-akamai-install-button@1x.png"/><img alt="Screenshot illustrating how to install Akamai DataStream in ContentKing" class="image" height="749" loading="lazy" src="/media/support/contentking-lfa-akamai-install-button@1x.png" srcset="/media/support/contentking-lfa-akamai-install-button@1x.png, /media/support/contentking-lfa-akamai-install-button@2x.png 2x" title="Install Akamai DataStream" width="1200"/></picture></div>
<ul>
<li>Specify in which region should the AWS S3 bucket be created (EU or US) and click on <code class="language-plaintext highlighter-rouge">Create bucket</code>
</li>
</ul>
<div class="media-container" style="max-width: 1200px;"><picture><source media="(max-width: 600px)" srcset="/media/support/contentking-lfa-akamai-region-bucket@1x.avif" type="image/avif"/><source srcset="/media/support/contentking-lfa-akamai-region-bucket@1x.avif, /media/support/contentking-lfa-akamai-region-bucket@2x.avif 2x" type="image/avif"/><source media="(max-width: 600px)" srcset="/media/support/contentking-lfa-akamai-region-bucket@1x.webp" type="image/webp"/><source srcset="/media/support/contentking-lfa-akamai-region-bucket@1x.webp, /media/support/contentking-lfa-akamai-region-bucket@2x.webp 2x" type="image/webp"/><source media="(max-width: 600px)" srcset="/media/support/contentking-lfa-akamai-region-bucket@1x.png"/><img alt="Screenshot illustrating Akamai DataStream credentials" class="image" height="543" loading="lazy" src="/media/support/contentking-lfa-akamai-region-bucket@1x.png" srcset="/media/support/contentking-lfa-akamai-region-bucket@1x.png, /media/support/contentking-lfa-akamai-region-bucket@2x.png 2x" title="Retrieve Akamai DataStream credentials" width="1200"/></picture></div>
<ul>
<li>After this, ContentKing will automatically generate the AWS credentials and the AWS S3 bucket</li>
</ul>
<div class="media-container" style="max-width: 1200px;"><picture><source media="(max-width: 600px)" srcset="/media/support/contentking-lfa-akamai-credentials@1x.avif" type="image/avif"/><source srcset="/media/support/contentking-lfa-akamai-credentials@1x.avif, /media/support/contentking-lfa-akamai-credentials@2x.avif 2x" type="image/avif"/><source media="(max-width: 600px)" srcset="/media/support/contentking-lfa-akamai-credentials@1x.webp" type="image/webp"/><source srcset="/media/support/contentking-lfa-akamai-credentials@1x.webp, /media/support/contentking-lfa-akamai-credentials@2x.webp 2x" type="image/webp"/><source media="(max-width: 600px)" srcset="/media/support/contentking-lfa-akamai-credentials@1x.png"/><img alt="Screenshot illustrating Akamai DataStream credentials" class="image" height="716" loading="lazy" src="/media/support/contentking-lfa-akamai-credentials@1x.png" srcset="/media/support/contentking-lfa-akamai-credentials@1x.png, /media/support/contentking-lfa-akamai-credentials@2x.png 2x" title="Retrieve Akamai DataStream credentials" width="1200"/></picture></div>
<ul>
<li>The credentials will be used to associate the AWS S3 bucket with the Akamai DataStream in the next section</li>
</ul>
<h3 class="toc" id="2-configuring-the-akamai-datastream">
<a aria-hidden="true" class="anchor-link" href="#2-configuring-the-akamai-datastream" tabindex="-1"></a>2. Configuring the Akamai DataStream</h3>
<p>The Akamai DataStream needs to be manually configured in the Akamai UI for every website. The configuration of the Akamai DataStream consists of two steps:</p>
<ol>
<li>Creating a new Akamai DataStream</li>
<li>Associating the DataStream to the corresponding property (domain)</li>
</ol>
<h3 id="creating-a-new-akamai-datastream">
<a aria-hidden="true" class="anchor-link" href="#creating-a-new-akamai-datastream" tabindex="-1"></a>Creating a new Akamai DataStream</h3>
<p>Follow these steps to create a new Akamai DataStream:</p>
<h5 id="1-go-to-datastream-section-in-your-akamai-account">1. Go to "DataStream" section in your Akamai account</h5>
<p>Log in to your <a href="https://control.akamai.com/apps/home-page/#/home" rel="noopener" target="_blank"><span class="extlink">Akamai Control Center</span><span class="show-for-sr"> (opens in a new tab)</span></a> and click on <code class="language-plaintext highlighter-rouge">DataStream</code> in the navbar:</p>
<div class="media-container" style="max-width: 1448px;"><img alt="Screenshot of the DataStream button in the Akamai UI navbar" class="image" height="1436" loading="lazy" src="/media/support/contentking-lfa-akamai-datastream-nav@2x.png" title="Akamai DataStream navbar" width="1448"/></div>
<h5 id="2-click-the-create-stream-button">2. Click the <code class="language-plaintext highlighter-rouge">Create stream</code> button</h5>
<p>On the "DataStream" screen, click <code class="language-plaintext highlighter-rouge">Create stream</code>:</p>
<div class="media-container" style="max-width: 2880px;"><img alt="Screenshot of the DataStream section in the Akamai UI" class="image" height="698" loading="lazy" src="/media/support/contentking-lfa-akamai-create-stream@2x.png" title="DataStream section in Akamai UI" width="2880"/></div>
<h5 id="3-configure-the-datastream">3. Configure the DataStream</h5>
<p>Specify the stream's name (e.g. "ContentKing LFA example.com") and pick the website for which you want to configure the Akamai DataStream in the Properties list:</p>
<div class="media-container" style="max-width: 2061px;"><img alt="Screenshot of the Create DataStream section in the Akamai UI" class="image" height="1560" loading="lazy" src="/media/support/contentking-lfa-akamai-stream-configuration@2x.png" title="Configuring a new DataStream" width="2061"/></div>
<h5 id="4-configure-the-data-sets">4. Configure the Data Sets</h5>
<p><strong>Important:</strong> The log format needs to be <strong>JSON</strong>.</p>
<p>Choose the following <em>required</em> data sets to be sent to the AWS S3 bucket:</p>
<ul>
<li>Log information
    <ul>
<li>Request ID</li>
<li>Request time</li>
</ul>
</li>
<li>Message exchange data
    <ul>
<li>Client IP</li>
<li>HTTP status code</li>
<li>Protocol type</li>
<li>Request host</li>
<li>Request method</li>
<li>Request path</li>
<li>Request port</li>
<li>Response-Content-Type</li>
<li>User-Agent</li>
<li>SSL version</li>
<li>Query string</li>
</ul>
</li>
</ul>
<div class="media-container" style="max-width: 2062px;"><img alt="Screenshot of the Data Sets step in creating an Akamai DataStream" class="image" height="1292" loading="lazy" src="/media/support/contentking-lfa-akamai-datasets@2x.png" title="Configuring Data Sets for the Akamai DataStream" width="2062"/></div>
<p>All the fields listed above need to be selected for the Log File Analysis feature to work properly.</p>
<p>You can also add other data sets, but the ones listed above are required.</p>
<section aria-label="Important!" class="highlighted-box typo-block highlighted-box--with-background"><div aria-hidden="true" class="highlighted-box__title">Important!</div>
<div class="highlighted-box__content">
<p>We require streaming only <em>non-PII</em> (non-personally identifiable information) data to our AWS S3.</p>
<p>To do that, you can filter away non-search engine traffic based on the User Agent string when setting up the DataStream directly in Akamai UI.</p>
<p>This is covered in the section <a href="#filter-away-non-search-engine-traffic">"Filter away non-search engine traffic"</a> below.</p>
</div></section>
<h5 id="5-define-a-destination">5. Define a destination</h5>
<ul>
<li>In the next step, choose "Amazon S3" as the destination where the data will be streamed.</li>
<li>Fill in the "Display name" (e.g. contentking-lfa-s3-example)</li>
<li>Enter the "Bucket name" generated by ContentKing to the "Bucket" field</li>
<li>Path (default suggested by Akamai):  <code class="language-plaintext highlighter-rouge">logs/{%Y}/{%m}/{%d}/{%H}</code>
</li>
<li>Enter the <em>"Region" that you selected in ContentKing</em>:
    <ul>
<li>If you've selected US: enter <code class="language-plaintext highlighter-rouge">us-east-2</code> (located in Ohio)</li>
<li>If you've selected EU: enter <code class="language-plaintext highlighter-rouge">eu-central-1</code> (located in Frankfurt)</li>
</ul>
</li>
<li>Enter the "Access key ID" and "Secret access key" from ContentKing</li>
<li>Once done, click on <code class="language-plaintext highlighter-rouge">Validate &amp; Save</code>
</li>
</ul>
<div class="media-container" style="max-width: 2062px;"><img alt="Screenshot of the Delivery step in creating an Akamai DataStream" class="image" height="1306" loading="lazy" src="/media/support/contentking-lfa-akamai-delivery@2x.png" title="Defining the logs destination" width="2062"/></div>
<h5 id="6-finish-the-setup">6. Finish the setup</h5>
<p>Finish the setup and click on the <code class="language-plaintext highlighter-rouge">Save stream</code> button:</p>
<div class="media-container" style="max-width: 838px;"><img alt="Screenshot of the Summary step in creating an Akamai DataStream" class="image" height="740" loading="lazy" src="/media/support/contentking-lfa-akamai-summary@2x.png" title="Summary screen of creating a stream" width="838"/></div>
<p><strong>Important:</strong> After configuring the DataStream, it might take several hours until Akamai sets everything up. Please <a href="/contact/">let us know</a> if it takes more than 24h.</p>
<h3 id="associate-the-created-datastream-to-the-corresponding-property">
<a aria-hidden="true" class="anchor-link" href="#associate-the-created-datastream-to-the-corresponding-property" tabindex="-1"></a>Associate the created DataStream to the corresponding property</h3>
<p>After creating the new DataStream, you need to associate it with the corresponding property in your Akamai account.</p>
<h5 id="1-go-to-properties-section-in-your-akamai-account">1. Go to "Properties" section in your Akamai account</h5>
<p>Log in to your <a href="https://control.akamai.com/apps/home-page/#/home" rel="noopener" target="_blank"><span class="extlink">Akamai Control Center</span><span class="show-for-sr"> (opens in a new tab)</span></a> and click on <code class="language-plaintext highlighter-rouge">Properties</code> in the navbar:</p>
<div class="media-container" style="max-width: 1506px;"><img alt="Screenshot of the Properties button in the Akamai UI navbar" class="image" height="1372" loading="lazy" src="/media/support/contentking-lfa-akamai-properties@2x.png" title="Akamai Properties navbar" width="1506"/></div>
<h5 id="2-choose-the-property">2. Choose the property</h5>
<p>Click the property which you want to associate with the created DataStream:</p>
<div class="media-container" style="max-width: 2046px;"><img alt="Screenshot of the Property Groups section in the Akamai UI" class="image" height="1078" loading="lazy" src="/media/support/contentking-lfa-akamai-property-groups@2x.png" title="Selecting a property to assosicate with the DataStream" width="2046"/></div>
<h5 id="3-choose-the-property-version">3. Choose the property version</h5>
<p>Click on the <em>Property Version</em> which is being monitored by ContentKing (this can be either the <em>Staging Version</em> or the <em>Production Version</em>):</p>
<div class="media-container" style="max-width: 2034px;"><img alt="Screenshot of the Property Details section in the Akamai UI" class="image" height="714" loading="lazy" src="/media/support/contentking-lfa-akamai-version@2x.png" title="Choosing a property version" width="2034"/></div>
<h5 id="4-scroll-down-to-the-datastream-section-in-the-property-configuration-settings">4. Scroll down to the "DataStream" section in the "Property Configuration Settings"</h5>
<div class="media-container" style="max-width: 1920px;"><img alt="Screenshot of the DataStream section in the Property Configuration Settings" class="image" height="598" loading="lazy" src="/media/support/contentking-lfa-akamai-datastream-section@2x.png" title="The DataStream section in the Property Configuration Settings" width="1920"/></div>
<p>If the "DataStream" section is missing, follow the steps below to add it. If not, skip to the <a href="#5-choose-the-stream-version">step 5</a>.</p>
<h5 id="41-scroll-all-the-way-down-and-click-edit-new-version">4.1 Scroll all the way down, and click "Edit new version"</h5>
<div class="media-container" style="max-width: 1950px;"><img alt="Screenshot of the Edit New Version button in the Property configuration Settings" class="image" height="796" loading="lazy" src="/media/support/contentking-lfa-akamai-edit-new-version@2x.png" title="Edit New Version button in the Property configuration Settings" width="1950"/></div>
<h5 id="42-scroll-down-to-the-property-configuration-settings-and-click-add-behavior">4.2 Scroll down to the "Property Configuration Settings", and click "Add Behavior"</h5>
<div class="media-container" style="max-width: 1948px;"><img alt="Screenshot of the Add Behavior button on the Cloudflare dashboard" class="image" height="766" loading="lazy" src="/media/support/contentking-lfa-akamai-addbehavior@2x.png" title="Add Behavior button in the Property configuration Settings" width="1948"/></div>
<h5 id="43-search-for-datastream-in-the-modal-that-appears-and-select-it">4.3 Search for "DataStream" in the modal that appears and select it</h5>
<div class="media-container" style="max-width: 1502px;"><img alt="Screenshot of DataStream in the behavior modal" class="image" height="962" loading="lazy" src="/media/support/contentking-lfa-akamai-datastream-behavior@2x.png" title="Searching for the DataStream in the behavior modal" width="1502"/></div>
<h5 id="44-click-insert-behavior">4.4 Click "Insert Behavior"</h5>

<div class="media-container" style="max-width: 1912px;"><img alt="Screenshot of selecting DataStream 2 as the stream version" class="image" height="674" loading="lazy" src="/media/support/contentking-lfa-akamai-datastream2@2x.png" title="Selecting DataStream 2 as the stream version" width="1912"/></div>
<h5 id="6-enable-the-datastream2-and-specify-the-name">6. Enable the DataStream2 and specify the name</h5>
<p>In the DataStream2 section, move the toggle to <code class="language-plaintext highlighter-rouge">On</code>, choose the Stream name of the new DataStream you created, and then click the <code class="language-plaintext highlighter-rouge">Save</code> button:</p>
<div class="media-container" style="max-width: 1918px;"><img alt="Screenshot of enabling and naming the DataStream 2" class="image" height="838" loading="lazy" src="/media/support/contentking-lfa-akamai-stream-name@2x.png" title="Enabling and naming the DataStream 2" width="1918"/></div>
<p><strong>Important:</strong> Once this is done, ALL the traffic on your website will be streamed to ContentKing's AWS S3 bucket. However, ContentKing will process only search engine visits, and filter out all other traffic, including all PII data.</p>
<p>Follow the next steps to learn how to filter away non-search engine traffic directly in the Akamai DataStream that you've created in the previous steps.</p>

<h5 id="2-scroll-to-the-log-request-details">2. Scroll to the "Log Request Details"</h5>

<div class="media-container" style="max-width: 1946px;"><img alt="Screenshot of the Loq Request Details in the Property Configuration Settings" class="image" height="1056" loading="lazy" src="/media/support/contentking-lfa-akamai-log-request-details@2x.png" title="Loq Request Details in the Property Configuration Settings" width="1946"/></div>
<p>In case the "Log Request Details" section is not there, follow the same steps as <a href="#4-scroll-down-to-the-datastream-section-in-the-property-configuration-settings">within step 4.</a> in the section "Associate the created DataStream to the corresponding property" to add it.</p>
<h5 id="3-enable-the-log-user-agent-header">3. Enable the "Log User-Agent header"</h5>

<div class="media-container" style="max-width: 1946px;"><img alt="Screenshot of the highlighted Log User-Agent header toggle" class="image" height="1056" loading="lazy" src="/media/support/contentking-lfa-akamai-log-ua-header@2x.png" title="Ensure that the Log User-Agent header is enabled" width="1946"/></div>
<h3 id="filter-away-non-search-engine-traffic">

<div class="media-container" style="max-width: 1980px;"><img alt="Screenshot of the Add Rule button in the Property Configuration Settings" class="image" height="754" loading="lazy" src="/media/support/contentking-lfa-akamai-add-rule@2x.png" title="Add Rule in the Property Configuration Settings" width="1980"/></div>

<div class="media-container" style="max-width: 1586px;"><img alt="Screenshot of the Blank Rule Template when adding a new rule" class="image" height="974" loading="lazy" src="/media/support/contentking-lfa-akamai-blank-rule-template@2x.png" title="Choosing a Blank Rule Template" width="1586"/></div>

<h3 id="reinstalling-the-akamai-datastream">
<a aria-hidden="true" class="anchor-link" href="#reinstalling-the-akamai-datastream" tabindex="-1"></a>Reinstalling the Akamai DataStream</h3>
<p>If you want to change the AWS S3 bucket region, or reinstall the Akamai DataStream, you can do it in the following way:</p>
<section class="highlighted-box typo-block highlighted-box--with-icon highlighted-box--steps-style"><div class="highlighted-box__content">
<ol>
<li>Click on the website on which you want to reinstall the Akamai DataStream in the <a href="https://app.contentkingapp.com/account/websites?view=list" rel="noopener" target="_blank"><span class="extlink">Websites section</span><span class="show-for-sr"> (opens in a new tab)</span></a> of ContentKing</li>
<li>Click on <code class="language-plaintext highlighter-rouge">Log File Analysis</code> in <code class="language-plaintext highlighter-rouge">Settings</code>, and then on the <code class="language-plaintext highlighter-rouge">reinstall</code> link under <code class="language-plaintext highlighter-rouge">Log Sources</code>
</li>
<li>If needed change the region, and click on <code class="language-plaintext highlighter-rouge">Create bucket</code>
</li>

</li>
</ol>

<h2 id="disabling-log-file-analysis">
<a aria-hidden="true" class="anchor-link" href="#disabling-log-file-analysis" tabindex="-1"></a>Disabling Log File Analysis</h2>
<p>Same as with enabling Log File Analysis, you need to disable the feature in ContentKing and then remove the Akamai DataStream in the Akamai UI.</p>
<h3 id="disabling-log-file-analysis-1">
<a aria-hidden="true" class="anchor-link" href="#disabling-log-file-analysis-1" tabindex="-1"></a>Disabling Log File Analysis</h3>
<section class="highlighted-box typo-block highlighted-box--with-icon highlighted-box--steps-style"><div class="highlighted-box__content">
<ol>
<li>Click on the website on which you want to disable Log File Analysis in the <a href="https://app.contentkingapp.com/account/websites?view=list" rel="noopener" target="_blank"><span class="extlink">Websites section</span><span class="show-for-sr"> (opens in a new tab)</span></a> of ContentKing</li>
<li>Click on the Log File Analysis in the Settings section</li>
<li>Move the Log File Analysis toggle to <code class="language-plaintext highlighter-rouge">Disabled</code> state</li>
</ol>
</div></section>
<p>Once this is done, ContentKing will automatically disable Akamai DataStream access to the AWS S3 bucket where the data was streamed.</p>
<h3 id="removing-the-akamai-datastream">
<a aria-hidden="true" class="anchor-link" href="#removing-the-akamai-datastream" tabindex="-1"></a>Removing the Akamai DataStream</h3>
<p>If you have disabled the Log File Analysis feature, you still need to remove the Akamai DataStream in your Akamai account.</p>
<p>This needs to be done manually, as ContentKing doesn't have access to your Akamai account.</p>

<h2 id="what-are-cookies">What are cookies?</h2>
<p>Cookies are small text files which are stored on the browser or hard drive of your Device when you visit a webpage or use an application. Cookies may be needed to show the webpage or application on your Device and are also used to enhance the user experience. Cookies cannot damage your Devices or the files saved on it. There are different types of cookies. Some cookies come directly from our website (first party cookies) and others come from third parties which place cookies on our site (third party cookies).</p>
<p>Cookies can be stored for varying lengths of time on your browser or device. Session cookies are deleted from your computer or device when you close your web-browser. Persistent cookies will remain stored on your computer or Device until deleted or until they reach their expiry date.</p>
<h2 id="how-do-we-use-cookies">How do we use cookies?</h2>
<h3 id="essential-technical-cookies">Essential (Technical) cookies</h3>
<p>We try to provide an advanced and user-friendly website that adapts automatically to needs and wishes of our visitors and users. To achieve this, we use technical cookies to i.e. show you our Website, to make it function correctly, to create your ContentKing account, to sign you in and to manage your requests. These technical cookies are necessary for our website to function properly.</p>
<h3 id="functional-cookies">Functional cookies</h3>
<p>We also use functional cookies to remember your preferences and settings (as username, password, language etc.) and to help you to use the Website efficiently and effectively. These functional cookies are not strictly necessary for the functioning of our Website, but they add functionality for you and enhance your experience.</p>
<h3 id="analytical-cookies">Analytical cookies</h3>
<p>We use analytical cookies in order to collect statistics about the use and visits of the Website. These analytic cookies generate statistical and other information about website use by means of cookies, which are stored on usersâ€™ Devices. The information generated relating to our Website is used to create reports about the use of the Website. Analytical cookies may be placed without your consent, unless such cookies have an impact on your privacy. In such cases, prior consent will be asked for.</p>
<h3 id="google-analytics">Google Analytics</h3>
<p>We use Google Analytics cookies in order to collect statistics about the use and visits of the Website. With these analytical cookies a permanent cookie is saved on your device to register your use of the Website. Google Analytics analyses this data and provides us with the results. This way, we obtain insight into the traffic of the Website and the way in which the Website is used. Based on this information we are able to make specific adjustments to the Website.</p>
<p>The information that we obtain is transferred to Google and stored by Google on servers outside the European Economic Area. We have entered into a data processing agreement with Google in which agreements have been made regarding the processing of personal data.</p>
<p>If you would like more information about analytical cookies and their expiry date, please visit the Google Analytics <a href="https://developers.google.com/analytics/devguides/collection/gajs/cookie-usage">information page</a>. Google also offers an <a href="https://tools.google.com/dlpage/gaoptout?hl=en">opt-out option</a> for data collection within the scope of Google Analytics.</p>
<h3 id="marketing-cookies">Marketing cookies</h3>
<p>We use third party marketing cookies in order to follow your internet browsing behavior and collect data and information on your browsing behavior from various websites you visited. This information is used to make the content of the displayed advertisements as relevant as possible, and to limit repetition of the same advertisements.</p>
<h3 id="social-plug-ins">Social plug-ins</h3>
<p>By using the Website you can get access to social media websites, such as Facebook, Twitter and LinkedIn. These buttons come with a code which has been made by these networks themselves. Using these so called social plug-ins you can login to your social media profile and subsequently share information from the Website with others. With these social plug-ins third party marketing cookies are saved on your Devices. These cookies serve the purpose of optimizing your user experience.</p>
<p>Companies such as LinkedIn, Twitter and Facebook may share your personal data outside the European Economic Area. Please read our Privacy Policy <a href="/legal/privacy-policy/">here</a> and the respective social networkâ€™s privacy statement to see how these companies treat your (personal) data.</p>
<h3 id="pixels">Pixels</h3>
<p>We also place pixels of third parties. A pixel keeps track of your surfing behavior and how you use the Website. These data are aggregated and give us information about the target group that visits our website. Based on this information, we can show an advertisement to a specific or similar target group on the website of the third party that produced the pixel.</p>

If you do not give consent, these cookies will not be placed. In that case, you may not be able to use all the functionalities on our Website.</p>
