<h1 class="heading__title">The SEO’s Guide to HTTP Status Codes</h1>

<section aria-label="HTTP Status Codes in Short" class="highlighted-box highlighted-box--with-background typo-block"><div aria-hidden="true" class="highlighted-box__title">
<span class="highlighted-box__title-icon"><i class="post-type-icon post-type-icon--reversed-style" style="background-image: url(/assets/images/icons/post-types/status-codes--reversed.svg)"></i></span>HTTP Status Codes in Short
	</div>
<div class="highlighted-box__content">
<p>HTTP status codes are three-digit responses a server returns at the request of a client (a browser or search engine). They are divided into five classes with multiple variants that convey different types of messages.</p>
<p>Every SEO out there should know at least the most common HTTP status codes by heart to quickly evaluate any situation:</p>
<ul>
<li>200 OK</li>
<li>301 Moved permanently</li>
<li>302 Found / Moved temporarily</li>
<li>404 Not Found</li>
<li>410 Gone</li>
<li>503 Service Unavailable</li>
</ul>

<h2 id="what-are-they">
<a aria-hidden="true" class="anchor-link" href="#what-are-they" tabindex="-1"></a>What are HTTP status codes?</h2>
<p>HTTP status codes are three-digit responses that a server returns at the request of a client (a browser or search engine). There are five classes of status codes, and each class conveys a different type of message.</p>
<p>When you requested this page in your browser, you made dozens of requests to our server asking for all of the information needed to view and render the page. That all happened in a flash, without you noticing. HTTP status codes were instrumental in making it possible.</p>
<p>HTTP stands for Hypertext Transfer Protocol. It's the protocol used by clients and servers to communicate and exchange data. HTTP status codes are part of the communication process.</p>
<p>Please note that we'll use the terms "URL," "page," and "resource" interchangeably, depending on the context. At the end of the day, these are all resources.</p>

<p>I personally see status code grooming as one of the most important elements of SEO hygiene and one of the most important ongoing processes.</p>
<p>Here's why it's important to understand HTTP status codes:</p>
<ul>
<li>We had many discussions over <code class="language-plaintext highlighter-rouge">301</code> vs <code class="language-plaintext highlighter-rouge">302</code> with some of the biggest disparity between what Google officially says versus what practicality tells us. I suggest taking care of any <code class="language-plaintext highlighter-rouge">302s</code> because even though they might pass PageRank in the same fashion as <code class="language-plaintext highlighter-rouge">301s</code>, Google will not update your snippet in the SERPs. <h2 id="timeouts">
<a aria-hidden="true" class="anchor-link" href="#timeouts" tabindex="-1"></a>A note on timeouts and other connection errors</h2>
<p>Timeout or connection errors unrelated to HTTP status code classes <strong>are not server responses</strong>. Errors like these mean that the client never reached the server.</p>
<p>As this is important to understand, but a confusing topic for most people, we'll look at a few examples:</p>
<ul>
<li>Connection timeout errors, such as Google Chrome's "This site can't be reached," are not status codes. The server wasn't reached, so they can't be server responses and aren't classified as status codes.</li>
<li>DNS lookup errors, such as Google Chrome's "This webpage is not available". This error isn't a server response either, as the client couldn't connect with the server because of a DNS issue.</li>
</ul>
<p>In all cases where you get a HTTP status code, this means the server was reached, and that it was able to issue a response.</p>
<h2 id="why-are-they-important-seo">
<a aria-hidden="true" class="anchor-link" href="#why-are-they-important-seo" tabindex="-1"></a>Why are HTTP status codes important to SEO?</h2>
<p>The goal of SEO is to drive organic traffic that's likely to be interested in what you have to say.</p>
<p>In order to drive that traffic, you need to make sure your content is accessible to search engine crawlers. When they request your content, you want HTTP status <code class="language-plaintext highlighter-rouge">200 OKs</code> to be returned. What you don't want is for HTTP status codes <code class="language-plaintext highlighter-rouge">5xx</code> and <code class="language-plaintext highlighter-rouge">4xx</code> to be returned, and you want to keep HTTP status codes <code class="language-plaintext highlighter-rouge">3xx</code> to a minimum as well.</p>
<p>Search engine crawlers and websites speak <code class="language-plaintext highlighter-rouge">HTTP</code>. If you're having a hard time understanding what they're talking about, how are you going to be an effective SEO?</p>
<p>Not understanding HTTP status codes is like managing a restaurant in a foreign country where you don't speak the language. You'll see lots of things happening, but you don't really know what's going on, why it happens, and how to improve it.</p>

<h2 id="what-do-http-responses-look-like">
<a aria-hidden="true" class="anchor-link" href="#what-do-http-responses-look-like" tabindex="-1"></a>What do HTTP responses look like?</h2>
<p>The <code class="language-plaintext highlighter-rouge">HTTP</code> responses sent by web servers usually consist of two parts: headers and a body payload.</p>
<p>The headers contain the HTTP status code and other information such as instructions on how long the client should cache the response.</p>
<p>The headers are not displayed to the user, but they do instruct the client on how to deal with the response and how to display the body payload (if any). You can see what <code class="language-plaintext highlighter-rouge">HTTP</code> headers were returned by using <a href="/academy/http-status-codes/#how-to-check">Web Inspector or other browser tools</a>.</p>
<p>When a body payload is present in the <code class="language-plaintext highlighter-rouge">HTTP</code> response, it's used to render and display the page to the user. When it comes to redirects, there is no body payload—and keep in mind that a body payload is optional for some errors.</p>
<h2 id="what-do-http-status-codes-look-like">
<a aria-hidden="true" class="anchor-link" href="#what-do-http-status-codes-look-like" tabindex="-1"></a>What do HTTP status codes look like?</h2>
<p>Let's look at a simplified example of what your browser did when you requested to see this page:</p>
<p><code class="language-plaintext highlighter-rouge">GET /academy/http-status-codes/ HTTP/2</code></p>
<p>This breaks down into:</p>
<ul>
<li>
<code class="language-plaintext highlighter-rouge">GET</code>: describes the <code class="language-plaintext highlighter-rouge">HTTP</code> method used to get what you want from the server.</li>
<li>
<code class="language-plaintext highlighter-rouge">/academy/http-status-codes/</code>: describes what URL the request is about.</li>
<li>
<code class="language-plaintext highlighter-rouge">HTTP/2</code>: describes what protocol to communicate in.</li>
</ul>
<p>And here's the HTTP response header our server sent back:</p>
<p><code class="language-plaintext highlighter-rouge">HTTP/2 200 OK</code></p>
<ul>
<li>
<code class="language-plaintext highlighter-rouge">HTTP/2</code> – describes what protocol to communicate in.</li>
<li>
<code class="language-plaintext highlighter-rouge">200 OK</code> – the request was successful—this is what you want to see.</li>
</ul>
<h2 id="common-status-codes">
<a aria-hidden="true" class="anchor-link" href="#common-status-codes" tabindex="-1"></a>What are the most common HTTP status codes in SEO?</h2>
<p>The most common HTTP status codes you'll come across in daily SEO life are:</p>
<ul>
<li><a href="/academy/http-status-codes/#200">200 OK</a></li>
<li><a href="/academy/http-status-codes/#301">301 Moved permanently</a></li>
<li><a href="/academy/http-status-codes/#302">302 Found / Moved temporarily</a></li>
<li><a href="/academy/http-status-codes/#307">307 Temporarily Redirect / Internal Redirect</a></li>
<li><a href="/academy/http-status-codes/#404">404 Not Found</a></li>
<li><a href="/academy/http-status-codes/#410">410 Gone</a></li>
<li><a href="/academy/http-status-codes/#503">503 Service Unavailable</a></li>
</ul>
<h2 id="how-to-check">
<a aria-hidden="true" class="anchor-link" href="#how-to-check" tabindex="-1"></a>How do you check HTTP status codes?</h2>
<p>You can see HTTP status codes right in your browser by using built-in tools, a browser plugin, or for instance <a href="/academy/url-inspection-tool/">Google Search Console's URL inspection tool</a>.</p>
<p>As these check types are all done on a page-by-page basis, they're slow and inefficient if you want to check a large number of URLs. They're even more tedious if you want to monitor those URLs. <p>When it comes to diagnose HTTP Status headers, one of the most common pitfalls I see, is when a server answers conditionally depending on the <code class="language-plaintext highlighter-rouge">User-Agent</code>.</p>
<p>For example, I've seen cases where a server replies with a <code class="language-plaintext highlighter-rouge">404</code> status, when a browser tries to access the robots.txt file, but when you use a stateless browser, or command line tool that emulates Googlebot, the server will reply something entirely different. The worst-case scenarios, is when in fact, there's an HTTP <code class="language-plaintext highlighter-rouge">500</code> status, instead of the <code class="language-plaintext highlighter-rouge">404</code> you saw in the browser.</p>

</code></pre></div></div>

<h2 id="different-types">
<a aria-hidden="true" class="anchor-link" href="#different-types" tabindex="-1"></a>The different types of HTTP statuses</h2>
<p>There are five classes of HTTP response status codes:</p>
<ul>
<li>
<a href="/academy/http-status-codes/#1xx">1xx – An informational response</a>:<br/>
"We've received your request and are processing it. Please hold!"</li>
<li>
<a href="/academy/http-status-codes/#2xx">2xx – A success response</a><br/>
"We've successfully received your request; here's what you asked for!"</li>
<li>
<a href="/academy/http-status-codes/#3xx">3xx – A redirection response</a><br/>
"Further action is needed to fulfill this request."</li>
<li>
<a href="/academy/http-status-codes/#4xx">4xx – A client error response</a><br/>
"The request cannot be fulfilled; something is probably wrong on your end."</li>
<li>
<a href="/academy/http-status-codes/#5xx">5xx – A server error response</a><br/>
"We're having issues processing your valid request."</li>
</ul>
<h3 class="toc" id="1xx">
<a aria-hidden="true" class="anchor-link" href="#1xx" tabindex="-1"></a>HTTP 1xx status codes</h3>
<p><code class="language-plaintext highlighter-rouge">HTTP 1xx</code> status codes are all informational. They indicate that a request was received and understood, but that it simply hasn't been processed yet.</p>
<p>We're not covering the <code class="language-plaintext highlighter-rouge">1xx</code> status codes, as they are only rarely seen.</p>

<h3 class="toc" id="2xx">
<a aria-hidden="true" class="anchor-link" href="#2xx" tabindex="-1"></a>HTTP 2xx status codes</h3>
<p><code class="language-plaintext highlighter-rouge">HTTP 2xx</code> status codes indicate successful requests. Everything went according to plan.</p>
<p>From an SEO point of view, the most important status code to know in the <code class="language-plaintext highlighter-rouge">2xx</code> range is the HTTP status code <code class="language-plaintext highlighter-rouge">200</code>.</p>
<h4 id="200">HTTP status code 200 OK</h4>
<h4 id="what-does-the-http-status-code-200-mean">What does the HTTP status code 200 mean?</h4>
<p>When a server sends the HTTP status code <code class="language-plaintext highlighter-rouge">200</code> as a response, it tells the client that the request was received successfully and replies with the requested content.</p>
<p>Please note that this is not an error code. The HTTP status <code class="language-plaintext highlighter-rouge">200 OK</code> means everything went OK!</p>
<h4 id="an-example-of-correct-use-of-the-http-status-code-200">An example of correct use of the HTTP status code 200</h4>
<p>When a page that exists is requested and this HTTP status code is the response.</p>
<h4 id="a-common-example-of-incorrect-use-of-the-http-status-code-200">A common example of incorrect use of the HTTP status code 200</h4>
<p>When a page that doesn't exist is requested, and an HTTP status code <code class="language-plaintext highlighter-rouge">200</code> is served instead of either an <a href="/academy/http-status-codes/#404">HTTP status code 404</a> or <a href="/academy/http-status-codes/#410">HTTP status code 410</a>. This is what we call a <a href="/academy/index-coverage/faq/submitted-soft-404/">soft 404 error</a>.</p>

<h3 class="toc" id="3xx">
<a aria-hidden="true" class="anchor-link" href="#3xx" tabindex="-1"></a>HTTP 3xx status codes</h3>
<p><code class="language-plaintext highlighter-rouge">HTTP 3xx</code> status codes indicate that further action is needed. The request was received, but can't be fulfilled yet. When the client receives a <code class="language-plaintext highlighter-rouge">3xx</code> status code, it needs to make a new request to the location the server returned along with the redirect. The HTTP status codes that we'll describe in this section are all redirects.</p>
<p>This happens when a request is made for URL A, but URL A is redirected to URL B. So the request should actually be made to URL B.</p>
<p>Redirects are like bandaids, you use them if you want to quickly fix something. Putting bandaids on top of each other doesn't work. Avoid doing the same with redirects. <a href="/academy/redirects/#avoid-chained-redirect">Don't chain redirects</a>; always redirect to the final destination of the redirect.</p>
<p>In this section we'll cover the following HTTP status codes, because of their significance for SEO:</p>
<ul>
<li><a href="/academy/http-status-codes/#301">301: Moved permanently</a></li>
<li><a href="/academy/http-status-codes/#302">302: Found / Moved temporarily</a></li>
<li><a href="/academy/http-status-codes/#303">303: See Other</a></li>
<li><a href="/academy/http-status-codes/#304">304: Not modified</a></li>
<li><a href="/academy/http-status-codes/#307">307: Temporarily Redirect</a></li>
<li><a href="/academy/http-status-codes/#308">308: Permanent Redirect</a></li>
</ul>

<h4 id="301">301 status code: Moved permanently</h4>
<h4 id="what-does-the-http-status-code-301-mean">What does the HTTP status code 301 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">301</code>, "301 redirect" for short, indicates that a resource has permanently moved to another location. The request and all future requests must be redirected to another URL instead of the requested source.</p>
<p>Because of its permanent nature, all signals from the requested URL that is being redirected are passed on to the destination URL. Please note that browsers will also cache 301 redirects to save an additional request in the future, which leads to a better user experience by displaying the page faster.</p>

<h4 id="an-example-of-correct-use-of-the-http-status-code-301">An example of correct use of the HTTP status code 301</h4>
<p>When a website has migrated from <code class="language-plaintext highlighter-rouge">HTTP</code> to <code class="language-plaintext highlighter-rouge">HTTPS</code>, and each <code class="language-plaintext highlighter-rouge">HTTP</code> URL is redirected to its <code class="language-plaintext highlighter-rouge">HTTPS</code> version. 301 redirects are essential to the success of website migrations.</p>
<h4 id="a-common-example-of-incorrect-use-of-the-http-status-code-301">A common example of incorrect use of the HTTP status code 301</h4>
<p>When a page is only temporarily inaccessible.</p>

<h3 id="302">
<a aria-hidden="true" class="anchor-link" href="#302" tabindex="-1"></a>302 status code: Found / Moved temporarily</h3>
<h4 id="what-does-the-http-status-code-302-mean">What does the HTTP status code 302 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">302</code>, "302 redirect" for short, indicates that a resource has temporarily moved to another location.</p>
<p>Because of its temporary nature, no popularity and relevance signals from the redirected URL are passed on to the destination URL and the redirected URL will still rank instead of being consolidated. Please note that after a significant amount of time, 302 redirects can be treated as 301 redirects by search engines.</p>
<p>Simply put: when a 302 redirect has been in place for a long time, it's treated as being permanent.</p>

<figcaption class="captioned-image__caption" id="image-caption-8">302 redirect: temporarily moved content</figcaption></figure>
<h4 id="an-example-of-correct-use-of-the-http-status-code-302">An example of correct use of the HTTP status code 302</h4>
<p>You're running a campaign, and for a short period of time, you want to send users landing on URL A to URL B instead. After three weeks the campaign ends, and you remove the 302 redirect.</p>
<h4 id="a-common-example-of-incorrect-use-of-the-http-status-code-302">A common example of incorrect use of the HTTP status code 302</h4>
<p>When a website migration is done, and 302 redirects are implemented instead of 301 redirects. Signals from old URLs aren't passed on to new URLs right away, so the new URLs will not be as successful as the old URLs. It can take a months before 302 redirects are considered to be actually 301 redirects.</p>

<h4 id="303">303 status code: See Other</h4>
<h4 id="what-does-the-http-status-code-303-mean">What does the HTTP status code 303 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">303</code>, "303 redirect" for short, indicates that the server is redirecting the request URL to a different URL. It can for instance be used to prevent users from accidentally re-submitting forms when using the "back" button in their browser, as the 303 redirect indicates that a follow-up request to the temporary URL should be made using the <code class="language-plaintext highlighter-rouge">GET</code> HTTP method.</p>
<p>When Google's Gary Illyes was asked whether 303 redirects pass popularity signals, <a href="https://twitter.com/methode/status/1118589457052119042" rel="noopener" target="_blank"><span class="extlink">he said "Yes"</span><span class="show-for-sr"> (opens in a new tab)</span></a>; it's fine to use 303 redirects when it comes to forms, but don't use them for redirecting content that isn't related to forms.</p>
<p>Why?</p>
<p>Because Google has noted that 303 redirects also pass link equity (similar to 301 redirects), but what they haven't noted is that it takes quite a while for this to happen.</p>
<div class="media-container" style="max-width: 1262px;"><img alt="Screenshot of Twitter exchange between Gary Illyes and Andy Kuiper whether 303 redirects pass link equity." class="image" height="532" loading="lazy" src="/wp-content/uploads/2019/06/contentking-http-status-codes-http-303-passes-link-equity-en.png" title="Screenshot of Twitter exchange between Gary Illyes and Andy Kuiper whether 303 redirects pass link equity." width="1262"/></div>
<h4 id="an-example-of-correct-use-of-the-http-status-code-303">An example of correct use of the HTTP status code 303</h4>
<p>You've got a contact form on your website, and you want to keep users from accidentally re-submitting their input when they use the "back" button in their browser.</p>
<h4 id="a-common-example-of-incorrect-use-of-the-http-status-code-303">A common example of incorrect use of the HTTP status code 303</h4>
<p>When content is permanently moved, and a 303 redirect is used instead of a 301 redirect.</p>

<h3 id="304">
<a aria-hidden="true" class="anchor-link" href="#304" tabindex="-1"></a>304 status codes: Not modified</h3>
<h4 id="what-does-the-http-status-code-304-mean">What does the HTTP status code 304 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">304</code>, "304 redirect" for short, indicates that the requested resource hasn't been modified since the last time it was requested; therefore it will not be returned to the client, and instead the cached version should be used.</p>
<p>Example: we've accessed a page on May 27th 2019 at 9:00 AM. On Thursday May 30th we're accessing the page again, but we're asking the server if the page has changed since the last time we accessed it. We can do so by adding this condition to the request.</p>
<p><code class="language-plaintext highlighter-rouge">If-Modified-Since: Mon, 27 May 2019 09:00:00 GMT</code></p>
<h4 id="an-example-of-correct-use-of-the-http-status-code-304">An example of correct use of the HTTP status code 304</h4>
<p>Serving the HTTP status code <code class="language-plaintext highlighter-rouge">304</code> to users when content hasn't changed is a best practice. From a search engine crawler point of view, you benefit most from the HTTP status code <code class="language-plaintext highlighter-rouge">304</code> with large websites, say 100,000 pages and up. When we're talking at this scale, it's really important to make sure search engine crawlers are <a href="/academy/crawl-budget/">efficiently using their crawl budget</a> for your website.</p>
<h4 id="a-common-example-of-incorrect-use-of-the-http-status-code-304">A common example of incorrect use of the HTTP status code 304</h4>
<p>Responding with an HTTP status code <code class="language-plaintext highlighter-rouge">304</code> when a page has actually changed.</p>

<h4 id="307">307 status code: Temporarily Redirect / Internal Redirect</h4>
<h4 id="what-does-the-http-status-code-307-mean">What does the HTTP status code 307 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">307</code>, "307 redirect" for short, is the HTTP 1.1 equivalent of <a href="/academy/http-status-codes/#302">the 302 redirect</a>. A 302 redirect doesn't guarantee that no changes are made to the HTTP method used, but a 307 redirect does. The 307 redirect was invented to make sure the HTTP method used to make a request doesn't change when the server responds with a redirect. For instance, if the <code class="language-plaintext highlighter-rouge">GET</code> HTTP method was used, then the <code class="language-plaintext highlighter-rouge">GET</code> HTTP method is passed on as part of the redirect.</p>
<h4 id="307-status-code-as-internal-redirect">307 status code as Internal Redirect</h4>
<p>The 307 redirect is also used as an internal redirect in cases where the browser knows <code class="language-plaintext highlighter-rouge">HTTPS</code> is enforced. The browser may know this either because it was told so when making previous requests, or because the domain is on the HSTS Preload list.</p>
<p>The HSTS Preload list is a list of domains that are using <code class="language-plaintext highlighter-rouge">HTTPS</code>. Even though a user may request the <code class="language-plaintext highlighter-rouge">HTTP</code> version of a URL, the browser will use the internal 307 redirect to request the <code class="language-plaintext highlighter-rouge">HTTPS</code> version of the URL. Doing so prevents unnecessary, unsafe requests.</p>
<p>This list is shared across browsers. See <a href="https://hstspreload.org/" rel="noopener" target="_blank"><span class="extlink">HSTS Preload List Submission</span><span class="show-for-sr"> (opens in a new tab)</span></a> for more information.</p>
<h4 id="an-example-of-correct-use-of-the-http-status-code-307">An example of correct use of the HTTP status code 307</h4>
<p>When a <code class="language-plaintext highlighter-rouge">307 Internal Redirect</code> is used to make sure a connection stays secure.</p>
<h4 id="a-common-example-of-incorrect-use-of-the-http-status-code-307">A common example of incorrect use of the HTTP status code 307</h4>
<p>When content is permanently moved, and a 307 redirect is used instead of a <a href="/academy/http-status-codes/#301">301 redirect</a>.</p>

<h4 id="308">308 status code: Permanent Redirect</h4>
<h4 id="what-does-the-http-status-code-308-mean">What does the HTTP status code 308 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">308</code>, "308 redirect" for short, is the HTTP 1.1 equivalent of the 301 redirect, and it does not allow changing the request method from <code class="language-plaintext highlighter-rouge">POST</code> to <code class="language-plaintext highlighter-rouge">GET</code>.</p>
<p>At the moment it's not very clear how much page authority the 308 redirect passes, so we recommend using the 301 redirect instead to indicate that content has moved permanently.</p>

<h3 class="toc" id="4xx">
<a aria-hidden="true" class="anchor-link" href="#4xx" tabindex="-1"></a>HTTP 4xx status codes</h3>
<p>HTTP <code class="language-plaintext highlighter-rouge">4xx</code> status codes indicate there was an error on the client side. Possible reasons for these errors are: the requesting party's not authorized to request a page, requesting nonexistent pages, and making too many requests.</p>

<h4 id="401">401 status code: Unauthorized</h4>
<h4 id="what-does-the-http-status-code-401-mean">What does the HTTP status code 401 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">401</code> is an error and describes that HTTP Authentication failed. The requested page requires either a username and password combination, and/or isn't allowed access based on its IP address.</p>
<p>You'll typically see the HTTP status <code class="language-plaintext highlighter-rouge">401</code> when crawlers try to access staging environments, and you've implemented HTTP Authentication to prevent this. An HTTP status code <code class="language-plaintext highlighter-rouge">401</code> is what you want to see; otherwise search engines could be indexing your staging environment, and third parties could be accessing privileged information.</p>
<h4 id="an-example-of-correct-use-of-the-http-status-code-401">An example of correct use of the HTTP status code 401</h4>
<p>When HTTP Authentication is implemented and you reject bad requests with a <code class="language-plaintext highlighter-rouge">401</code> HTTP status code.</p>
<h4 id="a-common-example-of-incorrect-use-of-the-http-status-code-401">A common example of incorrect use of the HTTP status code 401</h4>
<p>Using the HTTP status code <code class="language-plaintext highlighter-rouge">401</code> instead of the <code class="language-plaintext highlighter-rouge">403</code> to indicate that authorization failed.</p>

<h4 id="403">403 status code: Forbidden</h4>
<h4 id="what-does-the-http-status-code-403-mean">What does the HTTP status code 403 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">403</code>, "403 error" for short, indicates that it's forbidden to request a URL. This status code is commonly used as a permanent measure to prevent (search engine) crawlers from making requests after misbehaving. It's also returned in case the client provides the wrong login credentials.</p>
<p>Please note that the HTTP status code <code class="language-plaintext highlighter-rouge">401</code> and <code class="language-plaintext highlighter-rouge">403</code> are used in different situations.</p>
<h4 id="an-example-of-correct-use-of-the-http-status-code-403">An example of correct use of the HTTP status code 403</h4>
<p>When rogue crawlers are requesting too many URLs on your site, and you respond with the HTTP status code <code class="language-plaintext highlighter-rouge">403</code> to stop them.</p>
<h4 id="a-common-example-of-incorrect-use-of-the-http-status-code-403">A common example of incorrect use of the HTTP status code 403</h4>
<p>Using the HTTP status code <code class="language-plaintext highlighter-rouge">403</code> instead of the <code class="language-plaintext highlighter-rouge">401</code> to indicate that a client isn't authorized to make requests.</p>

<h4 id="404">404 status code: Not Found</h4>
<h4 id="what-does-the-http-status-code-404-mean">What does the HTTP status code 404 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">404</code>, "404 error" for short, indicates that the requested resource couldn't be found. It doesn't explicitly say it could be found before, unlike the HTTP status code <code class="language-plaintext highlighter-rouge">410</code>, but it just says: "Hey, we don't have what you're looking for (anymore).''</p>
<p>The <code class="language-plaintext highlighter-rouge">404</code> status code is probably the most common status code for most people browsing the web, as it's the one they see when clicking a broken link. For example, when the website owner removes a page, but there are still pages linking to it. You could say the returned <code class="language-plaintext highlighter-rouge">404</code> is not the actual problem—it's the page linking to it.</p>
<p>404 errors can also incorrectly occur for still existing content, due to bugs introduced by developers or maintenance being done on the web server. Having URLs return 404 errors is bad <strong>if you didn't remove the content</strong>.</p>
<p>While search engines won't drop URLs returning 404 errors from their index after hitting a <code class="language-plaintext highlighter-rouge">404</code> once, they may do so after a few tries, so you need to monitor for 404 errors. If a URL consistently returns errors, why would they keep it in their index?</p>
<p>Search engines aside, 404 errors are bad from a user-experience point of view as well, as they create a dead end for users. Keep tabs on them!</p>
<h4 id="an-example-of-correct-use-of-the-http-status-code-404">An example of correct use of the HTTP status code 404</h4>
<p>When you serve the HTTP status code <code class="language-plaintext highlighter-rouge">404</code> for all URLs that are requested that do not exist and have never existed.</p>
<h4 id="a-common-example-of-incorrect-use-of-the-http-status-code-404">A common example of incorrect use of the HTTP status code 404</h4>
<p>Using the HTTP status code <code class="language-plaintext highlighter-rouge">404</code> on pages that exist and are actually working fine.</p>

<h4 id="410">410 status code: Gone</h4>
<h4 id="what-does-the-http-status-code-410-mean">What does the HTTP status code 410 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">410</code>, "410 error" for short, indicates that the requested URL was permanently removed. This means the URL existed before, but it was explicitly removed and will not come back.</p>
<p>Search engines are quick to remove URLs from their index when the 410 error is returned. This makes it a powerful tool in any SEO's toolbox, so harness its power—with care.</p>
<h4 id="an-example-of-correct-use-of-the-http-status-code-410">An example of correct use of the HTTP status code 410</h4>
<p>When you serve the HTTP status code <code class="language-plaintext highlighter-rouge">410</code> for all URLs that were explicitly removed and will not come back.</p>

<p>Using the HTTP status code <code class="language-plaintext highlighter-rouge">410</code> for URLs that are temporarily unavailable.</p>

<h4 id="418">418 status code: I'm a teapot</h4>

<h4 id="429">429 status code: Too many requests</h4>
<h4 id="what-does-the-http-status-code-429-mean">What does the HTTP status code 429 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">429</code>, "429 error" for short, indicates that a client has been making too many requests to a server within a certain timeframe.</p>
<p>The 429 error can be seen as a temporary version of the <a href="/academy/http-status-codes/#403">HTTP status code 403</a>, provided we're talking about clients making too many requests. If the requests keep on coming in despite the 429 error being served, the server may decide to start responding with an HTTP status code <code class="language-plaintext highlighter-rouge">403</code> instead.</p>
<p>If the server keeps returning <code class="language-plaintext highlighter-rouge">429</code> HTTP status codes to Google for a longer period of time, Google may remove the content that they're receiving <code class="language-plaintext highlighter-rouge">429</code> HTTP status codes for. All the more reason to make sure your hosting platform is solid, and to always return the right status codes for the right job.</p>

<h4 id="an-example-of-correct-use-of-the-http-status-code-429">An example of correct use of the HTTP status code 429</h4>
<p>When a client requests too many URLs within a certain timeframe, and with an HTTP status code <code class="language-plaintext highlighter-rouge">429</code>, the server sends a clear signal that the client should slow down.</p>
<h4 id="a-common-example-of-incorrect-use-of-the-http-status-code-429">A common example of incorrect use of the HTTP status code 429</h4>
<p>Using the HTTP status code <code class="language-plaintext highlighter-rouge">429</code>, instead of the HTTP status code <code class="language-plaintext highlighter-rouge">403</code>, to fight off rogue crawlers.</p>

<h4 id="430">430 status code: Request Header Fields Too Large (Shopify)</h4>
<h4 id="what-does-the-http-status-code-430-mean">What does the HTTP status code 430 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">430</code> is an unassigned (unofficial) status code. This HTTP status code shouldn't be used.</p>
<p>However, Shopify has mixed up their use of status codes, and incorrectly sends a <code class="language-plaintext highlighter-rouge">430</code> status code when they should actually be sending a <code class="language-plaintext highlighter-rouge">429</code>.</p>
<p>So when you're monitoring a Shopify website, and you're receiving 430 errors, that means you're requesting too many URLs within a certain time frame. In this case, it's best to <a href="/support/crawl-speed/#setting-crawlspeed">decrease your monitoring speed</a>.</p>
<h4 id="an-example-of-correct-use-of-the-http-status-code-430">An example of correct use of the HTTP status code 430</h4>
<p>There is no correct use of this status code.</p>
<h4 id="a-common-example-of-incorrect-use-of-the-http-status-code-430">A common example of incorrect use of the HTTP status code 430</h4>
<p>Shopify using the HTTP status code <code class="language-plaintext highlighter-rouge">430</code> instead of <code class="language-plaintext highlighter-rouge">429</code>.</p>

<h4 id="451">451 status code: Unavailable for Legal Reasons</h4>
<h4 id="what-does-the-http-status-code-451-mean">What does the HTTP status code 451 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">451</code>, "451 error" for short, indicates that the requested URL isn't available due to legal reasons. It could for instance be served when you've been ordered (or just received a demand) to take down a certain page. The status code number <code class="language-plaintext highlighter-rouge">451</code> is a reference to the <a href="https://en.wikipedia.org/wiki/Fahrenheit_451" rel="noopener" target="_blank"><span class="extlink">Fahrenheit 451 book</span><span class="show-for-sr"> (opens in a new tab)</span></a>.</p>
<p>ISPs can also intervene and respond with an HTTP status code <code class="language-plaintext highlighter-rouge">451</code>, which happens in a lot of countries when illegal torrent sites are requested.</p>
<p>When this response is received by the client, an explanation should be provided as well, detailing why the resource isn't available, what legislation or regulation applies, and what it really applies to.</p>
<h4 id="an-example-of-correct-use-of-the-http-status-code-451">An example of correct use of the HTTP status code 451</h4>
<p>When someone has blatantly copied content from another website and was subsequently ordered to remove it and thus returns 451 errors to clients that request the URL, while explaining what happened.</p>
<h4 id="a-common-example-of-incorrect-use-of-the-http-status-code-451">A common example of incorrect use of the HTTP status code 451</h4>
<p>Serving 451 errors when people from the European Economic Area (EEA) are requesting resources from websites outside of the EEA, due to fears of GDPR repercussions.</p>

<h3 class="toc" id="5xx">
<a aria-hidden="true" class="anchor-link" href="#5xx" tabindex="-1"></a>HTTP 5xx status codes</h3>
<p><code class="language-plaintext highlighter-rouge">5xx</code> HTTP status codes, "<a href="/academy/http-status-codes/faq/server-error-5xx/">5xx errors</a>" for short, are served when clients make valid requests, but servers can't complete them for whatever reason. The website could for instance be too busy or temporarily down for maintenance.</p>
<p>If search engines often receive <code class="language-plaintext highlighter-rouge">5xx</code> errors (or <code class="language-plaintext highlighter-rouge">429</code> errors for that matter) when crawling a website, they may do a number of things—none of which are great from an SEO point of view:</p>
<ol>
<li>They may think the server can't handle this many requests, so they'll slow down crawling. This essentially means they're assigning <a href="/academy/crawl-budget/#definition-crawl-budget">less crawl budget</a>.</li>
<li>They can also demote pages, or completely remove them from the index, as they think the issues are persistent. Search engines aim to provide users with a good user experience, and—similar to <code class="language-plaintext highlighter-rouge">4xx</code> errors—<code class="language-plaintext highlighter-rouge">5xx</code> errors are the opposite of that.</li>
</ol>
<p>In this section we'll be covering the following status codes:</p>
<ul>
<li><a href="/academy/http-status-codes/#500">500: Internal Server Error</a></li>
<li><a href="/academy/http-status-codes/#503">503: Service Unavailable</a></li>
<li><a href="/academy/http-status-codes/#524">524: A timeout occurred</a></li>
</ul>
<h4 id="500">500 status code: Internal Server Error</h4>
<h4 id="what-does-the-http-status-code-500-mean">What does the HTTP status code 500 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">500</code>, "500 error" for short, indicates that the server had issues processing the request, but isn't able to explicitly state what went wrong.</p>
<h4 id="an-example-of-correct-use-of-the-http-status-code-500">An example of correct use of the HTTP status code <code class="language-plaintext highlighter-rouge">500</code>
</h4>
<p>When an unexpected error cropped up while processing the request, and no other <code class="language-plaintext highlighter-rouge">5xx</code> error applies.</p>
<h4 id="a-common-example-of-incorrect-use-of-the-http-status-code-500">A common example of incorrect use of the HTTP status code <code class="language-plaintext highlighter-rouge">500</code>
</h4>
<p>Serving a <code class="language-plaintext highlighter-rouge">500</code> error when the server is actually aware of the issue and should have responded with a more specific hint as to what's wrong.</p>

<h4 id="503">503 status code: Service Unavailable</h4>
<h4 id="what-does-the-http-status-code-503-mean">What does the HTTP status code 503 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">503</code>, "503 error" for short, indicates that the server is temporarily unavailable, and will be available again later. This can be due to scheduled maintenance (although we strongly recommend against doing this) or when the server is too busy.</p>
<p>The 503 error allows the inclusion of a "<code class="language-plaintext highlighter-rouge">Retry-After</code>" value in its response, basically saying: "Try again at a later time, I'll be able to process your request then".</p>
<p>Similar to the <a href="/academy/http-status-codes/#429">HTTP status code 429</a>, Google may remove content for which they're receiving <code class="language-plaintext highlighter-rouge">503</code> HTTP status codes for a longer period of time.</p>

<p>When the server is too busy, and can't process the client's request at this moment in time.</p>

<p>If your site goes down for any reason and pages become unavailable for an extended period of time, search engines will eventually remove the page from their index. To prevent this, use the <code class="language-plaintext highlighter-rouge">503</code> status code with the <code class="language-plaintext highlighter-rouge">Retry-After</code> header with a specified time, telling crawlers to try accessing the page again after an X amount of time. <h4 id="524">524 status code: A timeout occurred (Cloudflare)</h4>
<h4 id="what-does-the-http-status-code-524-mean">What does the HTTP status code 524 mean?</h4>
<p>The HTTP status code <code class="language-plaintext highlighter-rouge">524</code> doesn't officially exist. It was made up by Cloudflare and is sent when the request's origin times out. The issue isn't on Cloudflare's end; instead, the server that Cloudflare is relying on is the issue.</p>
<p>It is understandable for Cloudflare to have invented their own HTTP status code here, because when HTTP status codes were designed, companies like Cloudflare didn't exist yet.</p>

<h2 id="closing-words">
<a aria-hidden="true" class="anchor-link" href="#closing-words" tabindex="-1"></a>Closing words</h2>
<p>HTTP status codes play a fundamental role in quickly getting a message across between clients and servers. It's important for every SEO to know the most common ones by heart, as it makes them much more effective in their work: it lets them diagnose issues significantly faster.</p>
<p>Here's the list of the most common HTTP status codes of all:</p>
<ul>
<li><code class="language-plaintext highlighter-rouge">200 OK</code></li>
<li><code class="language-plaintext highlighter-rouge">301 Moved permanently</code></li>
<li><code class="language-plaintext highlighter-rouge">302 Found / Moved temporarily</code></li>
<li><code class="language-plaintext highlighter-rouge">404 Not Found</code></li>
<li><code class="language-plaintext highlighter-rouge">410 Gone</code></li>
<li><code class="language-plaintext highlighter-rouge">503 Service Unavailable</code></li>
</ul>
<p>Learn them well!</p>

<p>Cookies are small text files which are stored on the browser or hard drive of your Device when you visit a webpage or use an application. Cookies may be needed to show the webpage or application on your Device and are also used to enhance the user experience. Cookies cannot damage your Devices or the files saved on it. There are different types of cookies. Some cookies come directly from our website (first party cookies) and others come from third parties which place cookies on our site (third party cookies).</p>
<p>Cookies can be stored for varying lengths of time on your browser or device. Session cookies are deleted from your computer or device when you close your web-browser. Persistent cookies will remain stored on your computer or Device until deleted or until they reach their expiry date.</p>
<h2 id="how-do-we-use-cookies">How do we use cookies?</h2>
<h3 id="essential-technical-cookies">Essential (Technical) cookies</h3>
<p>We try to provide an advanced and user-friendly website that adapts automatically to needs and wishes of our visitors and users. To achieve this, we use technical cookies to i.e. show you our Website, to make it function correctly, to create your ContentKing account, to sign you in and to manage your requests. These technical cookies are necessary for our website to function properly.</p>
<h3 id="functional-cookies">Functional cookies</h3>
<p>We also use functional cookies to remember your preferences and settings (as username, password, language etc.) and to help you to use the Website efficiently and effectively. These functional cookies are not strictly necessary for the functioning of our Website, but they add functionality for you and enhance your experience.</p>
<h3 id="analytical-cookies">Analytical cookies</h3>
<p>We use analytical cookies in order to collect statistics about the use and visits of the Website. These analytic cookies generate statistical and other information about website use by means of cookies, which are stored on users’ Devices. The information generated relating to our Website is used to create reports about the use of the Website. Analytical cookies may be placed without your consent, unless such cookies have an impact on your privacy. In such cases, prior consent will be asked for.</p>
<h3 id="google-analytics">Google Analytics</h3>
<p>We use Google Analytics cookies in order to collect statistics about the use and visits of the Website. With these analytical cookies a permanent cookie is saved on your device to register your use of the Website. Google Analytics analyses this data and provides us with the results. This way, we obtain insight into the traffic of the Website and the way in which the Website is used. Based on this information we are able to make specific adjustments to the Website.</p>
<p>The information that we obtain is transferred to Google and stored by Google on servers outside the European Economic Area. We have entered into a data processing agreement with Google in which agreements have been made regarding the processing of personal data.</p>
<p>If you would like more information about analytical cookies and their expiry date, please visit the Google Analytics <a href="https://developers.google.com/analytics/devguides/collection/gajs/cookie-usage">information page</a>. Google also offers an <a href="https://tools.google.com/dlpage/gaoptout?hl=en">opt-out option</a> for data collection within the scope of Google Analytics.</p>
<h3 id="marketing-cookies">Marketing cookies</h3>
<p>We use third party marketing cookies in order to follow your internet browsing behavior and collect data and information on your browsing behavior from various websites you visited. This information is used to make the content of the displayed advertisements as relevant as possible, and to limit repetition of the same advertisements.</p>
<h3 id="social-plug-ins">Social plug-ins</h3>
<p>By using the Website you can get access to social media websites, such as Facebook, Twitter and LinkedIn. These buttons come with a code which has been made by these networks themselves. Using these so called social plug-ins you can login to your social media profile and subsequently share information from the Website with others. With these social plug-ins third party marketing cookies are saved on your Devices. These cookies serve the purpose of optimizing your user experience.</p>
<p>Companies such as LinkedIn, Twitter and Facebook may share your personal data outside the European Economic Area. Please read our Privacy Policy <a href="/legal/privacy-policy/">here</a> and the respective social network’s privacy statement to see how these companies treat your (personal) data.</p>
<h3 id="pixels">Pixels</h3>
<p>We also place pixels of third parties. A pixel keeps track of your surfing behavior and how you use the Website. These data are aggregated and give us information about the target group that visits our website. Based on this information, we can show an advertisement to a specific or similar target group on the website of the third party that produced the pixel.</p>
<h2 id="disable-or-delete-cookies">Disable or delete cookies</h2>
<p>When you first visit the Website, we ask you to give consent for cookies that are not necessary or functional and that have an impact on your privacy. If you do not give consent, these cookies will not be placed. In that case, you may not be able to use all the functionalities on our Website.</p>
<p>You can configure your browser so that you do not receive any cookies the next time you use the website. You can read here how you can remove different types of cookies in different browsers. Please consult the help-function of your browser if your browser is not listed below.</p>
<ul>
<li><a href="https://support.google.com/chrome/answer/95647?co=GENIE.Platform%3DDesktop&amp;hl=en-GB">Chrome</a></li>
<li><a href="https://support.mozilla.org/en-US/kb/clear-cookies-and-site-data-firefox">Firefox</a></li>
<li><a href="https://support.microsoft.com/en-us/microsoft-edge/delete-cookies-in-microsoft-edge-63947406-40ac-c3b8-57b9-2a946a29ae09">Edge</a></li>
<li><a href="https://support.apple.com/guide/safari/manage-cookies-and-website-data-sfri11471/mac">Safari</a></li>
</ul>
<p>Note: Refusing or deleting cookies only affects the Device and the browser on which you perform this action. If you use different Devices and/or browsers you will need to repeat the above described actions on these Devices and/or browsers.</p>
