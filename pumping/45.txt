<span class="highlighted-box__title-icon"><i class="post-type-icon post-type-icon--reversed-style" style="background-image: url(/assets/images/icons/post-types/architecture--reversed.svg)"></i></span>Shopware SEO in short

<div class="highlighted-box__content">

<h2 id="what-does-it-take">
<a aria-hidden="true" class="anchor-link" href="#what-does-it-take" tabindex="-1"></a>What does it take to rank your Shopware store well?</h2>
<p>Before we can answer that question, it’s important to understand how SEO works.</p>
<p>In a nutshell, search engine crawlers visit websites and try to find as much content as they can. They try to figure out what that content is about and whether they need to show it in search results. Not all content is equal though. Some content is more popular, warranting a higher position in their search results.</p>
<p>You can break that down into the “three pillars of SEO”:</p>
<ol>
<li>
<strong>Technology:</strong> the technical foundation of a website, which helps search engines find and understand your site as quickly and efficiently as possible. This is about setting yourself up for success.</li>
<li>
<strong>Relevance:</strong> what search queries your content is relevant for. This is done by creating content that’s useful and satisfying for your visitors. In order to communicate relevancy clearly, good technology is essential.</li>
<li>
<strong>Authority/Popularity:</strong> links to your website, which determine your authority and trustworthiness.</li>
</ol>
<p>This article zooms in on two of the pillars, <em>technology</em> and <em>relevance</em>, as these are platform specific, while <em>authority</em> is the same for every platform.</p>
<p>To read up on authority, check out our articles <a href="/academy/ecommerce-link-building/">Link building for eCommerce websites</a> and <a href="/academy/authority/">Authority and Trust in SEO explained</a>.</p>

<h2 id="technical-foundation">
<a aria-hidden="true" class="anchor-link" href="#technical-foundation" tabindex="-1"></a>A strong technical foundation for SEO performance</h2>
<p>This section explains in detail how to configure your Shopware platform for maximum SEO performance. The Shopware platform needs to be running on a solid hosting environment as well of course. We're using the <a href="https://www.hypernode.com/shopware-cloud-hosting/" rel="noopener" target="_blank"><span class="extlink">Hypernode hosting platform</span><span class="show-for-sr"> (opens in a new tab)</span></a> for our Shopware test environment because of their support, performance and scalability.</p>
<h3 id="url-structure">
<a aria-hidden="true" class="anchor-link" href="#url-structure" tabindex="-1"></a>URL structure</h3>
<p>A good URL structure is one that works for both visitors and search engines. So keep your URL structure:</p>
<ul>
<li>Descriptive and readable</li>
<li>Brief</li>
<li>Consistent</li>
<li>Lowercase</li>
</ul>
<p>With this in mind, log on to your Shopware backend and navigate to your URL structure settings:</p>
<ol>
<li>Default location: <code class="language-plaintext highlighter-rouge">https://domain.com/backend/</code>
</li>
<li>Go to <code class="language-plaintext highlighter-rouge">Configuration</code> &gt; <code class="language-plaintext highlighter-rouge">Basic settings</code>
</li>
<li>Open the folder <code class="language-plaintext highlighter-rouge">Frontend</code> and navigate to <code class="language-plaintext highlighter-rouge">SEO / router settings</code>
</li>
</ol>
<p>You’ll end up on this screen:</p>

<h4 id="general-url-structure-settings">General URL structure settings</h4>
<p>There, check your settings and change them if needed:</p>
<ul>
<li>The <code class="language-plaintext highlighter-rouge">Only use lower case letters in URLs</code> field should be set to <code class="language-plaintext highlighter-rouge">Yes</code> (it’s set correctly by default). This makes sure automatically created internal links are always lower case, and when URLs with uppercase letters are requested, they are canonicalized to their lowercase equivalent.</li>
<li>Set <code class="language-plaintext highlighter-rouge">Remove Category ID from URL</code> to <code class="language-plaintext highlighter-rouge">Yes</code> if needed. <strong>It is incorrectly set to</strong> <code class="language-plaintext highlighter-rouge">No</code> <strong>by default</strong>. Setting it to <code class="language-plaintext highlighter-rouge">Yes</code> removes the <code class="language-plaintext highlighter-rouge">?c=XX</code> variables that are otherwise added to product URLs.</li>
</ul>
<h4 id="product-category-url-settings">Product category URL settings</h4>
<p>The field that determines the product category URL structure is <code class="language-plaintext highlighter-rouge">SEO URLs category template</code>, and it’s set to <code class="language-plaintext highlighter-rouge">{sCategoryPath categoryID=$sCategory.id}/</code> by default.</p>
<p>This leads to a URL like <code class="language-plaintext highlighter-rouge">https://domain.com/category/</code>, or in the case of a subcategory, like <code class="language-plaintext highlighter-rouge">https://domain.com/category/subcategory/</code>.</p>
<p>We’re fans of having a trailing backslash at the end of each URL, but if you don’t want that, you could change the value to <code class="language-plaintext highlighter-rouge">{sCategoryPath categoryID=$sCategory.id}</code> to remove it.</p>
<h4 id="product-url-settings">Product URL settings</h4>
<p>The field that determines the product URL structure is <code class="language-plaintext highlighter-rouge">SEO URLs item template</code>, and it’s set to <code class="language-plaintext highlighter-rouge">{sCategoryPath articleID=$sArticle.id}/{$sArticle.id}/{$sArticle.name}</code> by default.</p>
<p>This leads to a URL like <code class="language-plaintext highlighter-rouge">https://domain.com/category/$number/product</code>, or in case of a subcategory <code class="language-plaintext highlighter-rouge">https://domain.com/category/subcategory/$number/product</code>.</p>
<p>Having <code class="language-plaintext highlighter-rouge">{$sArticle.id}</code> in the URL is useful if you have say 5,000 or more products. If you have fewer products, and their names are always different, then it’s fine to leave out the number, keeping your URLs shorter and descriptive.</p>
<p>As mentioned, we’re fans of having a trailing backslash at the end of each URL, so that’s why we’d recommend changing the value to <code class="language-plaintext highlighter-rouge">{sCategoryPath articleID=$sArticle.id}/{$sArticle.id}/{$sArticle.name}/</code>, so to add it.</p>
<h4 id="manufacturer-url-settings">Manufacturer URL settings</h4>
<p>We highly recommend using manufacturer-specific pages to increase the relevance of your product detail pages for a manufacturer’s brand name, and to target specific search queries built around the brand name itself.</p>
<section aria-label="Important" class="highlighted-box typo-block highlighted-box--with-background"><div aria-hidden="true" class="highlighted-box__title">Important</div>
<div class="highlighted-box__content">
<p>Please note that the terms “<em>manufacturer</em>” and “<em>supplier</em>” are used interchangeable in Shopware. We’ll call it manufacturer, because a supplier isn’t necessarily the manufacturer of a product.</p>
</div></section>
<p>In Shopware, the field that determines the manufacturer URL structure is <code class="language-plaintext highlighter-rouge">Supplier SEO URLs template</code> and it’s set to <code class="language-plaintext highlighter-rouge">{createSupplierPath supplierID=$sSupplier.id}/</code> by default.</p>
<p>This leads to a URL like <code class="language-plaintext highlighter-rouge">https://domain.com/manufacturer/</code>.</p>

<h4 id="blog-article-url-settings">Blog article URL settings</h4>
<p>If you’re going to be using the blogging functionality on your Shopware store, you want to make sure your blog articles have a good URL structure.</p>
<p>The field that determines the blog article URL structure is <code class="language-plaintext highlighter-rouge">SEO URLs blog template</code>, and it’s set to <code class="language-plaintext highlighter-rouge">{sCategoryPath categoryID=$blogArticle.categoryId}/{$blogArticle.title}</code> by default.</p>
<p>This leads to a URL structure like <code class="language-plaintext highlighter-rouge">https://domain.com/blog/category/article</code>.</p>
<p>To add the trailing slash, just adjust the value in the field to <code class="language-plaintext highlighter-rouge">{sCategoryPath categoryID=$blogArticle.categoryId}/{$blogArticle.title}/</code>.</p>
<p>If you don’t want to put your articles in a subfolder, you can change that by adjusting the value to <code class="language-plaintext highlighter-rouge">{$blogArticle.title}/</code>, leading to a URL like <code class="language-plaintext highlighter-rouge">https://domain.com/blog/article/</code>.</p>
<section aria-label="Important" class="highlighted-box typo-block highlighted-box--with-background"><div aria-hidden="true" class="highlighted-box__title">Important</div>
<div class="highlighted-box__content">
<p>By default, blog-article URLs don’t have a trailing slash, while pages in the shop section do have one. <strong>This is inconsistent, and you need to address it.</strong> Having a good, consistent URL structure is important both for both SEO and UX.</p>
</div></section>
<h4 id="shop-pages-url-settings">Shop pages URL settings</h4>
<p>You can define the default URL structure for shop pages and for regular pages such as your “about us” and “terms and conditions” pages.</p>
<p>The field that determines the URL structure for these pages is <code class="language-plaintext highlighter-rouge">Custom site SEO URLs template</code> and it’s set to <code class="language-plaintext highlighter-rouge">{$site.description}</code> by default.</p>
<p>To add the trailing slash, just adjust the value to <code class="language-plaintext highlighter-rouge">{$site.description}/</code>.</p>
<h4 id="form-pages-url-settings">Form pages URL settings</h4>
<p>You can define the default URL structure for pages that are centered on forms, such as a contact page.</p>
<p>The field that determines the URL structure for these pages is <code class="language-plaintext highlighter-rouge">Form SEO URLs template</code>, and it’s set to <code class="language-plaintext highlighter-rouge">{$form.name}</code> by default.</p>
<p>To add the trailing slash, just adjust the value to <code class="language-plaintext highlighter-rouge">{$form.name}/</code>.</p>
<h4 id="campaign-url-settings">Campaign URL settings</h4>
<p>Let’s say you’re running a Black Friday campaign. You can create that campaign under e.g. <code class="language-plaintext highlighter-rouge">Marketing</code> &gt; <code class="language-plaintext highlighter-rouge">Shopping Worlds</code>. The campaign’s URL is determined by the</p>
<p><code class="language-plaintext highlighter-rouge">SEO URLs landing page template</code> field, and it’s set to <code class="language-plaintext highlighter-rouge">{$campaign.name}</code> by default.</p>
<p>Again, to add the trailing slash, just define the value with <code class="language-plaintext highlighter-rouge">{$campaign.name}/</code>.</p>
<h4 id="shopware-automatically-rewrites-special-characters">Shopware automatically rewrites special characters</h4>
<p>Shopware uses the <a href="https://github.com/cocur/slugify" rel="noopener" target="_blank"><span class="extlink">Slugify framework</span><span class="show-for-sr"> (opens in a new tab)</span></a> to rewrite special characters such as ö, è, and ř. We recommend leaving this untouched if that works fine for your target audience, because it keeps things simple. For example, if you want to check whether your canonical link is the same as the URL you’re on, you can do that quickly if you’re letting Slugify rewrite special characters.</p>
<p>However, if you don’t want to Shopware to do this, you can <a href="https://developers.shopware.com/developers-guide/plugin-slugify/" rel="noopener" target="_blank"><span class="extlink">customize the Slugify ruleset</span><span class="show-for-sr"> (opens in a new tab)</span></a>.</p>

<h3 id="meta-information-title-and-meta-description">
<a aria-hidden="true" class="anchor-link" href="#meta-information-title-and-meta-description" tabindex="-1"></a>Meta information: title and meta description</h3>
<p>Your page’s title and meta description heavily influence its rankings, both directly and indirectly. What does this mean? Using important keywords in your title will help to rank your pages for those keywords. And once your pages are ranking, you want users to click on your snippet instead of your competitors’. Together with the URL, the title and meta description heavily influence <a href="/academy/ux-seo/#winning-clicks">whether or not a user will click on your snippet</a>. Think of them as your elevator pitch.</p>
<figure aria-labelledby="image-caption-5" class="captioned-image"><div class="media-container" style="max-width: 600px;"><picture><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-serp-snippet-en@1x.avif" type="image/avif"/><source srcset="/media/content/contentking-shopware-serp-snippet-en@1x.avif, /media/content/contentking-shopware-serp-snippet-en@2x.avif 2x" type="image/avif"/><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-serp-snippet-en@1x.webp" type="image/webp"/><source srcset="/media/content/contentking-shopware-serp-snippet-en@1x.webp, /media/content/contentking-shopware-serp-snippet-en@2x.webp 2x" type="image/webp"/><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-serp-snippet-en@1x.png"/><img alt="Screenshot of a search engine result snippet’s title and meta description" class="image" height="158" loading="lazy" src="/media/content/contentking-shopware-serp-snippet-en@1x.png" srcset="/media/content/contentking-shopware-serp-snippet-en@1x.png, /media/content/contentking-shopware-serp-snippet-en@2x.png 2x" title="A search engine result snippet from Google, pointing out the title and meta description" width="600"/></picture></div>
<figcaption class="captioned-image__caption" id="image-caption-5">A search engine result snippet from Google, pointing out the title and meta description</figcaption></figure>
<p>It’s thus important to think carefully about what you put into your titles and meta descriptions. You can save a lot of time by defining smart defaults for them. In most cases, these should lead to a good title and meta description, and if you want to overwrite them because it’s a really important page, you can do so, as described in the next section.</p>
<h4 id="defining-smart-defaults-for-titles">Defining smart defaults for titles</h4>
<p>It’s not possible to define title templates in Shopware out of the box; the way titles are built up is defined in the template as follows:</p>
<h5 id="product-categories-and-product-detail-pages">Product categories and product detail pages</h5>
<ul>
<li>Category pages: <code class="language-plaintext highlighter-rouge">$categoryName | $shopName</code>, so for example: “Superfoods | Your healthy food store”.</li>
<li>Subcategory pages: <code class="language-plaintext highlighter-rouge">$subcategoryName | $categoryName | $shopName</code>, so for example: “Nuts | Superfoods | Your healthy food store”.</li>
<li>Product detail pages: <code class="language-plaintext highlighter-rouge">$productName | $subcategoryName | $shopName</code>, so for example: “Almonds | Nuts | Your healthy food store”.</li>
</ul>
<h5 id="blog-category-and-blog-article-pages">Blog category and blog article pages</h5>
<ul>
<li>Blog category pages: <code class="language-plaintext highlighter-rouge">$blogCategoryName | $shopName</code>, so for example: “Healthy food news | Your healthy food store”.</li>
<li>Blog article pages: <code class="language-plaintext highlighter-rouge">$articleName | $blogCategoryName | $shopName</code>, so for example: “These nuts are nuts! | Healthy food news | Your healthy food store”.</li>
</ul>
<h5 id="other-pages">Other pages</h5>
<ul>
<li>CMS, form, manufacturer or campaign pages: <code class="language-plaintext highlighter-rouge">$pageName | $shopName</code>, so for example: “About us | Your healthy food store”.</li>
</ul>
<p>We recommend changing these titles to shorten them, because with the default structure they can easily become too long. This leads to search engines abbreviating your titles and to titles that lack focus and thus have less value for SEO.</p>
<p>You can <a href="https://docs.shopware.com/en/shopware-5-en/settings/seo?category=shopware-5-en/settings#individual-seo-title-for-category-blog-and-product-sites" rel="noopener" target="_blank"><span class="extlink">change the title templates in the code</span><span class="show-for-sr"> (opens in a new tab)</span></a>, or <a href="https://store.shopware.com/en/search?sSearch=meta%20information" rel="noopener" target="_blank"><span class="extlink">use a plugin</span><span class="show-for-sr"> (opens in a new tab)</span></a> for this.</p>

<li>Open the <code class="language-plaintext highlighter-rouge">Frontend</code> folder and navigate to <code class="language-plaintext highlighter-rouge">SEO / router settings</code>
</li>
</ol>

<h3 id="heading-structure">
<a aria-hidden="true" class="anchor-link" href="#heading-structure" tabindex="-1"></a>Heading structure</h3>
<p>A good heading structure helps visitors quickly scan a page and helps search engines easily understand its structure and topic.</p>
<p>Here are the best practices when it comes to the heading structure:</p>
<ul>
<li>Don’t wrap logos or images in <a href="/academy/headings/faq/h1/">H1 heading tags</a>, as the H1 heading conveys the main topic of the page.</li>
<li>Every page should have an H1 heading; use only one H1 heading per page.</li>
<li>For readability, sticking to a maximum heading length of 60 characters works well.</li>
<li>Build up your heading structure with a logical hierarchy, as illustrated below.</li>
</ul>

<h4 id="themes-dominate-your-heading-structure">Themes dominate your heading structure</h4>
<p>The correctness of your heading structure will largely be determined by the theme you’re using – not Shopware. For instance, if the theme wraps the product name into an H2 heading instead of an H1 heading, that needs to be addressed in the theme’s code, since there are no settings in Shopware to fix this.</p>

<p>To make you life easier, you should turn on the <code class="language-plaintext highlighter-rouge">Extended editor</code>, which lets you choose the headings when editing content, rather than having to use the source editor. <h3 id="navigation">
<a aria-hidden="true" class="anchor-link" href="#navigation" tabindex="-1"></a>Navigation</h3>
<p>Navigation plays a big role in every site’s SEO performance, and so that goes for your Shopware store as well.</p>
<p>The best practice is to only include the site’s most important pages in its main navigation, to keep from spreading the authority that flows through its internal links too thin.</p>

<h3 id="crawling-indexing">
<a aria-hidden="true" class="anchor-link" href="#crawling-indexing" tabindex="-1"></a>Crawling &amp; indexing: communicating your preferences</h3>
<p>In this section we'll cover what options you have in Shopware to communicate your preferences to search engines when it comes to crawling and indexing.</p>
<h4 id="canonical-urls">Canonical URLs</h4>
<p>Aside from automatic lowercasing, there are no settings surrounding the usage of canonical URLs available in Shopware. Out of the box, this works well.</p>
<h4 id="meta-robots--x-robots-tag">Meta robots / X-Robots-Tag</h4>
<p>Robots directives communicate your preferences concerning crawling and indexing, so they’re a powerful tool from an SEO point of view.</p>
<h5 id="query-parameter-handling-in-shopware">Query parameter handling in Shopware</h5>
<p>In Shopware, <code class="language-plaintext highlighter-rouge">meta robots noindex</code> is added by default to, for example, pages showing filtered or sorted products that have query parameters. This is correct. Query parameters are for instance used for filtering and sorting products.</p>
<p>For example, the URL <code class="language-plaintext highlighter-rouge">https://domain.com/nuts/almonds?sSupplier=gonuts</code> showing all products from the manufacturer “Gonuts” would have a meta robots noindex tag because it contains the parameter <code class="language-plaintext highlighter-rouge">sSupplier</code>, and that’s listed as one of the query parameters as seen below.</p>
<p>To manage which query parameters get this directive, go to the same screen where we defined the URL structure:</p>
<ol>
<li>
<code class="language-plaintext highlighter-rouge">Configuration</code> &gt; <code class="language-plaintext highlighter-rouge">Basic settings</code>
</li>
<li>Open the <code class="language-plaintext highlighter-rouge">Frontend</code> folder and navigate to <code class="language-plaintext highlighter-rouge">SEO / router settings</code>
</li>
</ol>
<p>You’ll end up on this screen:</p>
<figure aria-labelledby="image-caption-16" class="captioned-image"><div class="media-container" style="max-width: 600px;"><picture><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-seo-router-settings-noindex-en@1x.avif" type="image/avif"/><source srcset="/media/content/contentking-shopware-seo-router-settings-noindex-en@1x.avif, /media/content/contentking-shopware-seo-router-settings-noindex-en@2x.avif 2x" type="image/avif"/><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-seo-router-settings-noindex-en@1x.webp" type="image/webp"/><source srcset="/media/content/contentking-shopware-seo-router-settings-noindex-en@1x.webp, /media/content/contentking-shopware-seo-router-settings-noindex-en@2x.webp 2x" type="image/webp"/><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-seo-router-settings-noindex-en@1x.png"/><img alt="Screenshot of Shopware’s robots noindex settings" class="image" height="333" loading="lazy" src="/media/content/contentking-shopware-seo-router-settings-noindex-en@1x.png" srcset="/media/content/contentking-shopware-seo-router-settings-noindex-en@1x.png, /media/content/contentking-shopware-seo-router-settings-noindex-en@2x.png 2x" title="Shopware’s robots noindex settings" width="600"/></picture></div>

<p>Look for the <code class="language-plaintext highlighter-rouge">SEO noindex queries</code> field.</p>
<h5 id="adding-meta-robots-noindex-to-other-pages">Adding meta robots noindex to other pages</h5>
<p>In the same screen as shown above, one field below <code class="language-plaintext highlighter-rouge">SEO noindex queries</code>, it says <code class="language-plaintext highlighter-rouge">SEO noindex viewports</code>. The <code class="language-plaintext highlighter-rouge">SEO noindex viewports</code> field lists pages that have the <code class="language-plaintext highlighter-rouge">meta robots noindex</code> added. When you create landing pages for say PPC campaigns, list them here to prevent search engines from indexing them.</p>
<h4 id="pagination-attributes-link-prevnext">Pagination attributes: link prev/next</h4>
<p>The <strong>link prev/next</strong> pagination attribute communicates the relationship among a series of paginated pages. While Google has announced that they haven’t used these “hints” to understand relationships, it can still be useful to use them for other search engines, and to aid search engines in discovering links more easily in general.</p>
<p>By default, Shopware has pagination attributes disabled, but we recommend enabling them for the reasons mentioned above, and due to the fact that if this setting is disabled, Shopware automatically adds a <a href="/academy/canonical/">canonical URL</a> to the first page in the series of page, which is against Google’s best practices. Enabling the setting adds the pagination attributes and removes the canonical link to the first page in the series.</p>
<p>To enable the pagination attributes:</p>
<ol>
<li>Head over to <code class="language-plaintext highlighter-rouge">Configuration</code> &gt; <code class="language-plaintext highlighter-rouge">Basic settings</code>
</li>
<li>Open the folder <code class="language-plaintext highlighter-rouge">Frontend</code> and navigate to <code class="language-plaintext highlighter-rouge">SEO / router settings</code>
</li>
</ol>
<p>There look for the field named <code class="language-plaintext highlighter-rouge">Use prev/next-tag on paginated sites</code> and set it to <code class="language-plaintext highlighter-rouge">Yes</code>:</p>
<figure aria-labelledby="image-caption-17" class="captioned-image"><div class="media-container" style="max-width: 600px;"><picture><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-seo-router-settings-pagination-en@1x.avif" type="image/avif"/><source srcset="/media/content/contentking-shopware-seo-router-settings-pagination-en@1x.avif, /media/content/contentking-shopware-seo-router-settings-pagination-en@2x.avif 2x" type="image/avif"/><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-seo-router-settings-pagination-en@1x.webp" type="image/webp"/><source srcset="/media/content/contentking-shopware-seo-router-settings-pagination-en@1x.webp, /media/content/contentking-shopware-seo-router-settings-pagination-en@2x.webp 2x" type="image/webp"/><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-seo-router-settings-pagination-en@1x.png"/><img alt="Screenshot of Shopware’s pagination settings" class="image" height="277" loading="lazy" src="/media/content/contentking-shopware-seo-router-settings-pagination-en@1x.png" srcset="/media/content/contentking-shopware-seo-router-settings-pagination-en@1x.png, /media/content/contentking-shopware-seo-router-settings-pagination-en@2x.png 2x" title="Shopware’s pagination settings" width="600"/></picture></div>

<section aria-label="Important" class="highlighted-box typo-block highlighted-box--with-background"><div aria-hidden="true" class="highlighted-box__title">Important</div>
<div class="highlighted-box__content">
<p>Fixing this issue introduces another one. On paginated pages, no self-referencing canonical is present. That needs to be addressed, because otherwise you could create <a href="/academy/crawler-traps/">a crawler trap</a>. This issue needs to be addressed within the store’s templates. When implementing this fix, follow <a href="/academy/pagination/#how-to-implement-the-pagination-attributes">these guidelines</a>.</p>

<h4 id="robotstxt">Robots.txt</h4>
<p>The robots.txt communicates the rules of engagement to crawlers. Here you can tell them which URLs or sections of your site not to touch.</p>
<p>Out of the box, Shopware comes with the following robots.txt, which keeps crawlers from accessing for example account and shopping pages, and references the XML sitemap.</p>
<section aria-label="A fragment of code" class="code-box"><div class="code-box__code">
<figure class="highlight"><pre><code class="language-text" data-lang="text">User-agent: *
Disallow: /compare/
Disallow: /checkout/
Disallow: /register/
Disallow: /account/
Disallow: /address/
Disallow: /note/
Disallow: /widgets/
Disallow: /listing/
Disallow: /ticket/
Sitemap: http://domain.com/sitemap_index.xml</code></pre></figure>
</div></section>
<p>Besides preventing search engines from accessing the pages above, we also recommend adding:</p>
<section aria-label="A fragment of code" class="code-box"><div class="code-box__code">
<figure class="highlight"><pre><code class="language-text" data-lang="text">Disallow: /search
Disallow: /*?sInquiry=</code></pre></figure>
</div></section>
<p>The first addition keeps your internal search result pages from being accidentally crawled and indexed. The second addition keeps search engines from accessing URLs that include what product an inquiry has been made for.</p>
<section aria-label="Pro-tip" class="highlighted-box typo-block highlighted-box--with-background"><div aria-hidden="true" class="highlighted-box__title">Pro-tip</div>
<div class="highlighted-box__content">
<p>For shops with more than 10,000 products, we also recommend disallowing URLs with the query parameters below, so as to preserve crawl budget:</p>
<ul>
<li><code class="language-plaintext highlighter-rouge">sPage</code></li>
<li><code class="language-plaintext highlighter-rouge">sPerPage</code></li>
<li><code class="language-plaintext highlighter-rouge">sSupplier</code></li>
<li><code class="language-plaintext highlighter-rouge">sFilterProperties</code></li>
<li><code class="language-plaintext highlighter-rouge">p</code></li>
<li><code class="language-plaintext highlighter-rouge">n</code></li>
<li><code class="language-plaintext highlighter-rouge">s</code></li>
<li><code class="language-plaintext highlighter-rouge">f</code></li>
</ul>
</div></section>

<h3 id="xml-sitemap">
<a aria-hidden="true" class="anchor-link" href="#xml-sitemap" tabindex="-1"></a>XML Sitemap</h3>
<p>An XML sitemap is a file that contains all the URLs you want search engines to visit and index. It’s an efficient way to make your preferences clear to search engines, so we recommend always using an XML sitemap.</p>
<p>Shopware automatically generates a gzip-compressed XML Sitemap, located here: <code class="language-plaintext highlighter-rouge">http://domain.com/sitemap_index.xml</code>.</p>
<p>These page types are automatically included if they’re set to <code class="language-plaintext highlighter-rouge">active</code>:</p>
<ul>
<li>Product category pages</li>
<li>Product detail pages</li>
<li>Blog category pages</li>
<li>Blog detail pages</li>
<li>Manufacturer pages</li>
</ul>
<p>The follow page types are not included:</p>
<ul>
<li>Regular pages (Shop pages and form pages)</li>
</ul>
<h4 id="making-changes-to-your-xml-sitemap">Making changes to your XML Sitemap</h4>
<p>Additionally, Shopware doesn’t support image XML Sitemaps out of the box, so we recommend <a href="https://store.shopware.com/en/search?sSearch=sitemap" rel="noopener" target="_blank"><span class="extlink">getting a plugin for this</span><span class="show-for-sr"> (opens in a new tab)</span></a> or, if you’re able to make changes yourself, <a href="https://docs.shopware.com/en/shopware-5-en/settings/seo?category=shopware-5-en/settings#extending-the-xml-template" rel="noopener" target="_blank"><span class="extlink">editing the source</span><span class="show-for-sr"> (opens in a new tab)</span></a>.</p>

<h3 id="mobile-friendliness">
<a aria-hidden="true" class="anchor-link" href="#mobile-friendliness" tabindex="-1"></a>Mobile-friendliness</h3>

<h2 id="relevance">
<a aria-hidden="true" class="anchor-link" href="#relevance" tabindex="-1"></a>Tweaking your content for maximum relevance</h2>
<p>In the “Meta information” and “Heading structure” sections above, we describe how to use smart templates to generate good titles, meta descriptions, and headings.</p>
<p>Unfortunately, you can’t really automate <strong>all</strong> of this – you’ll find that for your most important pages, you’ll want to manually overwrite the values your templates generated.</p>
<p>Good news: Shopware supports this out of the box!</p>
<p>In this section we’ll show you how to manually overwrite the values automatically generated for your titles, meta descriptions, and headings.</p>
<h3 id="product-category-pages">
<a aria-hidden="true" class="anchor-link" href="#product-category-pages" tabindex="-1"></a>Product category pages</h3>
<p>To overwrite the generated values for product category pages, go to: <code class="language-plaintext highlighter-rouge">Items</code> &gt; <code class="language-plaintext highlighter-rouge">Categories</code>.</p>
<p>When you click on a product category, you’ll see all of its settings in a new screen. Scroll all the way down to the <code class="language-plaintext highlighter-rouge">CMS functions</code> and <code class="language-plaintext highlighter-rouge">Meta information</code> sections:</p>
<figure aria-labelledby="image-caption-22" class="captioned-image"><div class="media-container" style="max-width: 600px;"><picture><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-overwrite-heading-meta-information-en@1x.avif" type="image/avif"/><source srcset="/media/content/contentking-shopware-overwrite-heading-meta-information-en@1x.avif, /media/content/contentking-shopware-overwrite-heading-meta-information-en@2x.avif 2x" type="image/avif"/><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-overwrite-heading-meta-information-en@1x.webp" type="image/webp"/><source srcset="/media/content/contentking-shopware-overwrite-heading-meta-information-en@1x.webp, /media/content/contentking-shopware-overwrite-heading-meta-information-en@2x.webp 2x" type="image/webp"/><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-overwrite-heading-meta-information-en@1x.png"/><img alt="Screenshot of the product category detail page showing header and meta information" class="image" height="345" loading="lazy" src="/media/content/contentking-shopware-overwrite-heading-meta-information-en@1x.png" srcset="/media/content/contentking-shopware-overwrite-heading-meta-information-en@1x.png, /media/content/contentking-shopware-overwrite-heading-meta-information-en@2x.png 2x" title="Managing headings and meta information for product categories" width="600"/></picture></div>
<figcaption class="captioned-image__caption" id="image-caption-22">Managing headings and meta information for product categories</figcaption></figure>
<p>By default, the fields <code class="language-plaintext highlighter-rouge">Header</code>, <code class="language-plaintext highlighter-rouge">Meta title</code> and <code class="language-plaintext highlighter-rouge">Meta description</code> are empty. Filling in their values will overwrite the automatically generated values.</p>
<p>Now you’ll be asking: “But what do I fill in here?”</p>
<p>The answer is: the result of your Keyword Research – the keywords you’re looking to target.</p>
<p>As a rule of thumb, stick to the following best practices:</p>
<ul>
<li>
<strong>Header:</strong> make sure it’s easy to read, include your most important keyword, and don’t make the header longer than 60 characters.</li>
<li>
<strong>Meta title:</strong> include your most important keyword for the product category and keep its length between 285 and 575 pixels (± 30 and 60 characters respectively).</li>
<li>
<strong>Meta description:</strong> include your most important keyword for the product category and keep its length between 430 and 920 pixels (± 70 and 155 characters respectively).</li>
</ul>
<p>The above rules of thumb also apply for the other page types mentioned below.</p>
<p>You don’t have to fill anything in for the <code class="language-plaintext highlighter-rouge">Meta keywords</code> field, as per our explanation here.</p>
<h3 id="product-detail-pages">
<a aria-hidden="true" class="anchor-link" href="#product-detail-pages" tabindex="-1"></a>Product detail pages</h3>
<p>To overwrite the values generated for product detail pages, go to: <code class="language-plaintext highlighter-rouge">Items</code> &gt; <code class="language-plaintext highlighter-rouge">Overview</code>.</p>
<p>Open the folder of a product’s product category, and then click the pencil icon:</p>

<h3 id="other-pages-1">

<p>Changing the H1 heading, title, and meta description for other page types such as: campaign pages, manufacturer pages, form pages, and blog articles works very similar. When you’re going over to edit these pages, you’ll find the same or nearly identical fields every time.</p>

<h2 id="other-seo-issues-to-address-and-be-mindful-of">
<a aria-hidden="true" class="anchor-link" href="#other-seo-issues-to-address-and-be-mindful-of" tabindex="-1"></a>Other SEO issues to address and be mindful of</h2>
<p>In this section we'll describe other SEO issues that you need to address and be mindful of.</p>
<h3 id="links-to-external-sites">
<a aria-hidden="true" class="anchor-link" href="#links-to-external-sites" tabindex="-1"></a>Links to external sites</h3>
<p>Shopware automatically adds the <a href="/academy/nofollow/">nofollow link attribute</a> to all links to external sites, unless they’re whitelisted using the <code class="language-plaintext highlighter-rouge">SEO follow backlinks</code>’ field (see screenshot below). In practice, this means that in a lot of cases, most links to external sites have the nofollow link attribute, sending the following message to search engines: “I do want to link to this other website, but I don’t trust them, so don’t credit them.”</p>
<p>This is incorrect, and we highly recommend disabling this feature altogether in your template files.</p>
<p>If this is not an option for whatever reason, you’ll have to do the hard work of configuring “whitelisted” sites through these steps:</p>
<ol>
<li>
<code class="language-plaintext highlighter-rouge">Configuration</code> &gt; <code class="language-plaintext highlighter-rouge">Basic settings</code>
</li>
<li>Open the folder <code class="language-plaintext highlighter-rouge">Frontend</code> and navigate to <code class="language-plaintext highlighter-rouge">SEO / router settings</code>
</li>
</ol>
<p>You’ll end up on this screen:</p>
<figure aria-labelledby="image-caption-28" class="captioned-image"><div class="media-container" style="max-width: 600px;"><picture><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-seo-router-settings-follow-backlinks-en@1x.avif" type="image/avif"/><source srcset="/media/content/contentking-shopware-seo-router-settings-follow-backlinks-en@1x.avif, /media/content/contentking-shopware-seo-router-settings-follow-backlinks-en@2x.avif 2x" type="image/avif"/><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-seo-router-settings-follow-backlinks-en@1x.webp" type="image/webp"/><source srcset="/media/content/contentking-shopware-seo-router-settings-follow-backlinks-en@1x.webp, /media/content/contentking-shopware-seo-router-settings-follow-backlinks-en@2x.webp 2x" type="image/webp"/><source media="(max-width: 600px)" srcset="/media/content/contentking-shopware-seo-router-settings-follow-backlinks-en@1x.png"/><img alt="Screenshot of Shopware’s settings about what external links shouldn’t be marked as nofollow" class="image" height="333" loading="lazy" src="/media/content/contentking-shopware-seo-router-settings-follow-backlinks-en@1x.png" srcset="/media/content/contentking-shopware-seo-router-settings-follow-backlinks-en@1x.png, /media/content/contentking-shopware-seo-router-settings-follow-backlinks-en@2x.png 2x" title="Managing what external links shouldn’t be marked as nofollow" width="600"/></picture></div>
<figcaption class="captioned-image__caption" id="image-caption-28">Managing what external links shouldn’t be marked as nofollow</figcaption></figure>
<p>Look for the <code class="language-plaintext highlighter-rouge">SEO follow backlinks</code> field . All domains listed there are excepted from getting the <code class="language-plaintext highlighter-rouge">nofollow</code> link attribute.</p>
<h3 id="meta-keywords">
<a aria-hidden="true" class="anchor-link" href="#meta-keywords" tabindex="-1"></a>Meta keywords</h3>
<p>In several places, Shopware lets you enter meta keywords.</p>
<p>Meta keywords haven’t been used by Google in over a decade, and we strongly believe other search engines haven’t been using them for a long, long time either.</p>
<p>Don’t bother entering keywords in the meta keywords field. Writing satisfying content for your visitors or gaining some links to your product category pages is a much better way to spend your time!</p>
<h3 id="setting-different-http-status-codes-for-pages-that-dont-exist">
<a aria-hidden="true" class="anchor-link" href="#setting-different-http-status-codes-for-pages-that-dont-exist" tabindex="-1"></a>Setting different HTTP status codes for pages that don’t exist</h3>
<p>When you go to</p>
<ol>
<li>
<code class="language-plaintext highlighter-rouge">Configuration</code> &gt; <code class="language-plaintext highlighter-rouge">Basic settings</code>
</li>
<li>Open the folder <code class="language-plaintext highlighter-rouge">Frontend</code> and navigate to <code class="language-plaintext highlighter-rouge">SEO / router settings</code>
</li>
</ol>
<p>Towards the bottom of the list you’ll see the field named <code class="language-plaintext highlighter-rouge">"Page not found" error code</code>. By default it’s set to <code class="language-plaintext highlighter-rouge">404</code>, and we highly recommend keeping it that way. That’s the only correct <a href="/academy/http-status-codes/">HTTP status code</a> for a page that never existed.</p>
<p>For pages that did exist in the past, you could, however, return a <a href="/academy/http-status-codes/#410">status code 410: Gone</a>, but that will require custom code. In most cases you won’t need it.</p>
<h3 id="separate-xml-sitemap-for-mobile-visitors">
<a aria-hidden="true" class="anchor-link" href="#separate-xml-sitemap-for-mobile-visitors" tabindex="-1"></a>Separate XML sitemap for mobile visitors</h3>
<p>Shopware’s own SEO guide mentions a separate XML sitemap for mobile visitors. A separate XML sitemap that for mobile visitors is only useful if you have a separate desktop and mobile site. If you have a design that automatically adjusts itself to the device being used to interact with it – a responsive design – you don’t need to pay attention to this.</p>
<h2 id="wrapping-it-up">
<a aria-hidden="true" class="anchor-link" href="#wrapping-it-up" tabindex="-1"></a>Wrapping it up</h2>
<p>This Shopware SEO guide should give you a head start in improving your SEO performance. Good luck, and be sure to check out <a href="/academy/ecommerce-link-building/">our guide on link building for eCommerce sites</a> too!</p>
